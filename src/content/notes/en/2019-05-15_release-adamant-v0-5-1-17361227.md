This is mostly a bug fix release. With AIP-14 now accepted and clients implementing it, the behavior was corrected to be consistent with the final AIP version. The release fixes bugs where the /api/chatrooms endpoint returned inconsistent data, and the endpoints now work according to AIP-14. Signal messages no longer appear in chatrooms.

### Breaking changes

The `withPayments` parameter was replaced with `withoutDirectTransfer`, which is opposite to it; direct transfers are now returned by default. The default limit for /api/chatrooms was decreased to 25 records from 100. All clients working with /api/chatrooms should be updated and use only version 0.5.1 or later.
