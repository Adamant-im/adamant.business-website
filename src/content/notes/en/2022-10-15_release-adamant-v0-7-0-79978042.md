This maintenance release for ADAMANT focuses on housekeeping and dependency hygiene. All project dependencies have been updated to their latest compatible versions, ensuring continued security and compatibility across the build pipeline.

Legacy code has been moved into a dedicated `/legacy` directory to keep the active source tree clean and make it easier for contributors to identify current versus deprecated modules. In addition, legacy documentation that no longer reflects the project's current architecture has been removed.

### Breaking changes

Developers or integrators who referenced legacy code at its previous location will need to update their import paths to point to the new `/legacy` directory. Similarly, any consumers relying on the removed legacy documentation should consult the current documentation set for up-to-date guidance.
