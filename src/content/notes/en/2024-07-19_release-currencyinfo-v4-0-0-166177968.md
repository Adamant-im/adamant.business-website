This release introduces a new `GET /status` endpoint for monitoring service health. The `/get` and `/getHistory` endpoints now include `version` and `last_update` fields in their responses. Notifications can be sent to ADAMANT Messenger and Discord when significant changes are detected. A Dockerfile has been added to simplify deployment.

### Breaking changes

A new Significant Changes Detecting System has been implemented to handle rate differences from multiple sources. Additionally, ExchangeRateHost now requires an API key to function, which may require configuration updates for existing deployments.
