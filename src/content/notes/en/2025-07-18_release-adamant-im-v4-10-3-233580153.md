ADAMANT Messenger version 4.10.3 introduces a split screen layout, a redesigned login screen, and an updated Android app icon. The update also adds vibrations for specific actions, a spinner during the restore process, and a show or hide function for password login. Message sending timeouts and handling for wallets with no connection or nodes have been implemented, along with a transforming message status icon that changes to a check once a transaction is registered in a node. A GitHub Workflow with testnet ADM nodes has been added for build and deployment.

Several bug fixes address wallet and transaction issues, including displaying a question mark instead of NaN, fixing balance and fiat estimates, generating local transaction IDs, and correcting pending transaction fees and failed transaction statuses. UI issues such as removing the window max width, fixing address dialog warnings, and resolving favicon errors have been resolved. Platform-specific fixes include Android download permissions and Electron Linux build mode. Authentication by QR code has also been fixed.

Technical improvements feature dependency updates, including updating Sass, replacing qs with native URLSearchParams, updating Vite to v6, and upgrading to Capacitor v7. The build system has switched from CommonJS to ECMAScript Modules, and APK builds for Android have been added. Hash routing for static hosting has been enabled, and self-hosting instructions have been added to the documentation. Translations have also been updated.
