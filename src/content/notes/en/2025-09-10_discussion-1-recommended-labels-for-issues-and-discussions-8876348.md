ADAMANT uses a unified label set across both Issues and Discussions. While some labels may not apply equally to both, sharing the full set gives the community flexibility. Default GitHub labels are kept lowercase, while custom labels start with a capital letter (excluding prefixes).

GitHub's built-in labels are retained as-is: `bug`, `documentation`, `duplicate`, `enhancement`, `good first issue`, `help wanted`, `invalid`, `question`, and `wontfix`.

## Special Statuses

Three custom status labels indicate priority and scope. `High priority` marks urgent issues needing immediate attention. `Primary feature` flags features essential to product functionality. `Composite task` identifies large or complex work items that break down into multiple subtasks.

## Languages, Frameworks, and Technologies

Technology labels track the relevant stack: `JavaScript` and `TypeScript` for frontend and backend code, `Vue` for Vue.js features, `CSS` for styling and theming, `NodeJS` for backend logic and APIs, `Swift` for iOS, `Kotlin` for Android, `IPFS` for decentralized storage, `Electron` for desktop packaging, and `Capacitor` for mobile builds and integrations.

## Product, Platform, and Workflow

Platform-specific labels include `Web`, `Mobile`, and `Desktop` for client-specific issues, while `UX/UI` covers interface and experience improvements. `Performance` targets speed, storage, or network optimizations. `Protocol & AIPs` covers ADM blockchain protocol changes and node interactions, and `Integration` handles external blockchain, service, or API connections. `Auth` encompasses passphrases, keys, passkeys, biometrics, and secure local storage. `Nodes` tracks ADM node software, APIs, connectivity, and consensus. `DB` covers PostgreSQL, MongoDB, migrations, and queries. `Infrastructure` spans coin and ADAMANT nodes, servers, VPSs, Ansible, Nginx, Docker, and Zabbix. `APIs` addresses API design and endpoints. `Automation`, `CI/CD`, and `Linter` cover scripts, GitHub Actions, pipelines, and code quality checks (ESLint/Stylelint).

## Discussion Labels

Discussion-specific labels help categorize conversations. `Guideline` covers best practices, coding standards, and contribution rules. `Blockchain` and `Cryptocurrency` address ledger mechanics and ADM or other token discussions. `Messaging` relates to messengers, communication protocols, and mentions. `Decentralization` and `Privacy` track efforts to reduce centralized dependencies and protect user anonymity. `Encryption` and `Security` cover end-to-end encryption, cryptographic algorithms, key management, and vulnerabilities. `Governance` handles AIPs and proposals. `Community`, `Ideas`, and `Research` support feedback, long-term roadmap suggestions, and investigation. `Trading` covers bots, markets, exchanges, and liquidity. `Memes` is for community fun content. `Marketing` and `Marketing offers` address promotion, community growth, partnerships, exchange listings, and KOL engagement.

Each custom label is paired with a color for visual identification in GitHub's interface.
