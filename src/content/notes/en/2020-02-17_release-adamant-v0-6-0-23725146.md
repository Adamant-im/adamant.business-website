This release enhances socket connections by returning the `recipientPublicKey` field. The Transactions API, including KVS and Chats, now returns `block_timestamp` alongside existing response data.

A fix for migrations has been applied, resolving issues that previously affected database upgrade paths. The `/states/get/` endpoint has been expanded to support `SenderIds` and `keyIds` parameters, and now also accepts POST requests, broadening the ways clients can query state information.

Documentation has been updated to reflect these changes.

### Breaking changes

No breaking changes are introduced in this release.
