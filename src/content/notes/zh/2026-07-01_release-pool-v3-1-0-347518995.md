ADAMANT Forging Pool v3.1.0 是推荐给矿池运营者的更新。

本次发布修复并强化了奖励计算和支付记账路径，将矿池存储迁移至由 MongoDB 支持的集合，并为旧版矿池数据添加了从 LowDB 到 MongoDB 的迁移工具。该版本引入了具备重试安全性的每位投票人奖励进度跟踪机制，以防止在崩溃或部分执行后产生重复奖励；在支付记账前对存储的待处理奖励进行归一化处理；并增加了可选的矿池密码加密支持。新的 `adm-pool` 运营者 CLI 提供了 `encrypt`、`unlock`、`lock` 和 `status` 命令，以及一个仅限本地所有者使用的控制套接字，用于运行时的锁/解锁操作。新增的 `/api/health` 端点支持无需密钥的外部监控。仪表板现已支持投票人和交易筛选、代表/投票人名称显示、数值排序修复以及链接优化。文档已更新，涵盖设置、配置、迁移、监控和运营流程。运行环境基线已升级至 Node.js 22.13.0+ 和 npm 10+。

这是一次推荐升级，因为它包含了针对奖励计算和计划支付的修复与保护措施。升级前，运营者应备份现有矿池配置和奖励历史，检查 `config.default.jsonc` 中的 MongoDB 配置，阅读 `README.md` 中的迁移指南，在现有数据副本上测试迁移流程，并在迁移后验证支付设置、捐赠设置和维护钱包设置。密码加密为可选功能，现有的明文密码配置仍受支持。运营者可使用以下命令加密代表密码：

```sh
npm run adm-pool encrypt
```

然后使用以下命令解锁正在运行的矿池：

```sh
npm run adm-pool unlock
```
