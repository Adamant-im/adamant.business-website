此版本引入了通用 macOS 构建，并将项目迁移到新的 ESLint 配置。仓库中已添加发布说明文件。健康检查系统已改进：在发起 HTTP 请求前会显示“正在更新…”状态，初始和后续健康检查采用优化后的请求算法，当无法访问网页域名时系统将检查 `alt_ip`。修复了系统休眠后后台健康检查无法恢复的问题，并解决了节点状态报告错误的问题。钱包页面已更新，同时更新了 adamant-wallets 的脚本，并为该工具创建了开发测试页面。

消息现在可以以气泡布局形式分组显示。应用整体进行了多项样式更新，包括消息输入框样式、导出密钥表单布局以及通用视觉一致性改进。ADAMANT 欢迎聊天现在包含一个已禁用的输入框。作为统一风格和现代化 UI/UX 的一部分，新增了仅用于开发的 Playwright 检查。

此版本包含大量错误修复。聊天中的日期现在可正确刷新。打开节点页面时不再显示错误的同步指示。修复了向自身地址发送 BTC、DOGE 和 DASH 时交易金额错误的问题。修复了回复包含长链接的消息时标记损坏的问题。解决了与 `maintainScrollPosition` 相关的空引用错误。静态聊天在重新登录后不再为只读状态。PWA 与 iOS 之间的 ERC20 币种交易费用现在保持一致。修复了 PWA 页面上的交易导航和本地化 UI 回归问题。对遗留的持久化钱包符号进行了规范化处理，以防止在 `/options` 路由上发生崩溃。已修正发布的 APK 名称，并解决了 ESLint 9 检测到的源代码问题。修复了仓库中错误的提交活动记录。

通用依赖项和工具已升级，并移除了构建警告。Node.js 从版本 20 升级至 22。已实施组织范围的 Release Drafter 和可复用工作流。新增了 GitHub Actions 工作流以及用于 ESLint 的 Husky postinstall git 钩子。添加了针对 PWA 的 AI 代理操作指南文档，并更新了 README 及发布元数据。

### 重大变更

钱包已完全移除对 Klayr (KLY) 的支持。持有 KLY 资产的用户请注意，该加密货币在 v4.11.0 及后续版本中不再受支持。
