俄罗斯联邦媒体监管机构 Roskomnadzor 尝试屏蔽 ADAMANT Messenger 的网站。根据 ADAMANT 团队的说法，这一行为恰恰凸显了去中心化架构的韧性，因为该平台的设计不存在可供针对的单一控制点。

## 为何对 ADAMANT 实施审查不切实际

ADAMANT Messenger 采用区块链技术，提供一个不依赖第三方基础设施的通信平台。与将数据存储在中心化服务器上的传统 messaging 应用不同，ADAMANT 将消息分布于由独立运营节点组成的网络中。该网络没有可关闭的中心服务器，也没有单一组织控制数据。

ADAMANT 团队回应直接明确："审查是不可能的。ADAMANT 是真正的去中心化协议。这不仅是一种宣称，而是我们平台架构中实际设计的功能。" 这些特性在平台的服务条款中有明确记录，该条款既可通过明网访问，也可通过 Tor 网络获取。

![Roskomnadzor 尝试审查 ADAMANT Messenger：对去中心化的一次考验](/images/engineering-notes/medium/c05711e8c865/004-0-xm12prccf1rsvrxa.webp)

## ADAMANT 基金会

Roskomnadzor 的请求中提到了名为 "ADAMANT TECH LABS" 的组织，但该组织已不复存在。自 2020 年 12 月起，ADAMANT 已作为基金会运作，无领导者或中心化管理结构。该开源项目由 GitHub 上的开发者社区共同维护。

![Roskomnadzor 尝试审查 ADAMANT Messenger：对去中心化的一次考验](/images/engineering-notes/medium/c05711e8c865/005-0-ptiedse-rssglttj.webp)

## 对俄罗斯用户的影响

即使官方网站被屏蔽，ADAMANT 协议仍能正常运行。由于消息存储在多个独立节点上，而非中心服务器，因此网络没有单一的可下线端点。此次事件成为对 ADAMANT 去中心化声明的一次现实压力测试，而系统在整个过程中保持运行。

俄罗斯用户应预期主网站、web 应用 URL 以及 `adamant.im` 下的公共节点可能无法访问。作为临时解决方案，用户可通过 VPN 或 Tor 网络连接。Windows、Linux 和 macOS 的桌面应用可从项目的 GitHub 发布页面获取，它们直接连接协议，无需依赖 web 前端。

作为长期解决方案，用户可在配置中启用 API 后运行自己的 ADAMANT 节点，从开源仓库部署自己的 Messenger 实例，并将 Messenger 的配置文件指向该自托管节点。这可彻底消除对任何可能被屏蔽的公共基础设施的依赖。

## 结论

Roskomnadzor 的屏蔽行为凸显了国家机构试图控制数字通信与旨在抵抗此类控制的去中心化平台之间的持续紧张关系。对 ADAMANT 而言，此次事件实际证明了基于区块链的 messaging 协议即使在主要网站无法访问的情况下仍能持续运行。ADAMANT 团队重申其致力于改进平台，确保全球范围内的安全、私密、无审查通信。
