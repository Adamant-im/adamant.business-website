ADAMANT 在 Issues 和 Discussions 中使用统一的标签集。尽管某些标签对两者不完全适用，但共享完整标签集可为社区提供灵活性。默认的 GitHub 标签保持小写，而自定义标签以大写字母开头（前缀除外）。

GitHub 内置标签原样保留：`bug`、`documentation`、`duplicate`、`enhancement`、`good first issue`、`help wanted`、`invalid`、`question` 和 `wontfix`。

## 特殊状态

三个自定义状态标签用于表示优先级和范围。`High priority` 标记需要立即关注的紧急问题。`Primary feature` 标记对产品功能至关重要的特性。`Composite task` 识别可分解为多个子任务的大型或复杂工作项。

## 语言、框架和技术

技术标签用于跟踪相关技术栈：`JavaScript` 和 `TypeScript` 用于前后端代码，`Vue` 用于 Vue.js 特性，`CSS` 用于样式和主题，`NodeJS` 用于后端逻辑和 API，`Swift` 用于 iOS，`Kotlin` 用于 Android，`IPFS` 用于去中心化存储，`Electron` 用于桌面端打包，`Capacitor` 用于移动端构建和集成。

## 产品、平台和工作流

平台特定标签包括用于客户端问题的 `Web`、`Mobile` 和 `Desktop`，而 `UX/UI` 涵盖界面和体验改进。`Performance` 针对速度、存储或网络优化。`Protocol & AIPs` 涉及 ADM 区块链协议变更和节点交互，`Integration` 处理外部区块链、服务或 API 连接。`Auth` 涵盖密码、密钥、通行密钥、生物识别和安全本地存储。`Nodes` 跟踪 ADM 节点软件、API、连接性和共识机制。`DB` 涵盖 PostgreSQL、MongoDB、迁移和查询。`Infrastructure` 涵盖代币和 ADAMANT 节点、服务器、VPS、Ansible、Nginx、Docker 和 Zabbix。`APIs` 涉及 API 设计和端点。`Automation`、`CI/CD` 和 `Linter` 涵盖脚本、GitHub Actions、流水线和代码质量检查（ESLint/Stylelint）。

## 讨论标签

特定于讨论的标签有助于对对话进行分类。`Guideline` 涵盖最佳实践、编码标准和贡献规则。`Blockchain` 和 `Cryptocurrency` 涉及账本机制以及 ADM 或其他代币的讨论。`Messaging` 与消息应用、通信协议和提及相关。`Decentralization` 和 `Privacy` 跟踪减少中心化依赖和保护用户匿名性的努力。`Encryption` 和 `Security` 涵盖端到端加密、加密算法、密钥管理以及漏洞问题。`Governance` 处理 AIP 和提案。`Community`、`Ideas` 和 `Research` 支持反馈、长期路线图建议和调研。`Trading` 涵盖交易机器人、市场、交易所和流动性。`Memes` 用于社区娱乐内容。`Marketing` 和 `Marketing offers` 涉及推广、社区增长、合作伙伴关系、交易所上线和 KOL 合作。
