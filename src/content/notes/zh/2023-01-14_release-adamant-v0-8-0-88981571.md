此版本的 ADAMANT 在 `config.json` 中引入了 `cors` 选项。用户现在可在 `/api/transactions` 和 `/api/chats/get` 端点中使用 `inId` 或 `isIn`。例如，`/api/chats/get?InId=U6386412615727665758` 和 `/api/chats/get?isIn=U6386412615727665758` 是等效的。

在 `/api/chats/get` 和 `/api/chatrooms` 端点中的用户 ID 现在不区分大小写。例如，`/api/chatrooms/U6386412615727665758` 和 `/api/chatrooms/u6386412615727665758` 是相同的。

交易以及 `POST /api/accounts/delegates` 现在接受作为纯对象或位于 `transaction` 属性内的交易。

```bash
curl -X POST https://endless.adamant.im/api/transactions/process
   -H 'Content-Type: application/json'
   -d '{"type": 0, "amount": 100000000, ...}'  # or {"transaction": { "type": 0, "amount": 100000000, ... }}
```

`/api/states/get` 端点现在包含 `confirmations` 属性。已应用 `generatorPublicKey` 查询优化。修复了“permission denied for schema public”错误，以及其他一些小改动。

### Breaking changes

`/api/blocks` 端点不再返回 `count` 属性。
