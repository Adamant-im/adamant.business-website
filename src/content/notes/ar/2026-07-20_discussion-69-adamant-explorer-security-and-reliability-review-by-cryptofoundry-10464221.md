أكمل ADAMANT Explorer مراجعة مركزة لأمان وموثوقية واجهة HTTP العامة، وحدود ADAMANT Node، وسلوك ذاكرة التخزين المؤقت Redis، ودورة حياة Socket.IO، والثقة في الوكيل العكسي، وعرض المتصفح. تم دمج التحسينات في [adamant-explorer#37](https://github.com/Adamant-im/adamant-explorer/pull/37) وتُغلق المشاكل [#23](https://github.com/Adamant-im/adamant-explorer/issues/23)، [#25](https://github.com/Adamant-im/adamant-explorer/issues/25)، و[#33](https://github.com/Adamant-im/adamant-explorer/issues/33). غطت المراجعة ترتيب وسائط Express، وتعرّض واجهة برمجة التطبيقات العامة، والتحقق من الصحة، وتحديد المعدل، والثقة في الوكيل العكسي، واستجابات ADAMANT Node كحدود بيانات غير موثوقة، وصحة ذاكرة التخزين المؤقت Redis وسلوكها عند الفشل، واستقصاء Socket.IO وإعادة الاتصال، وعرض المتصفح للقيم التي تتحكم فيها Node والأقران، وفشل التبعيات الاختيارية، واستمرارية أسعار الصرف، وتقارير الصحة التشغيلية، ونمذجة التهديدات في المستودع.

## حدود HTTP العامة وواجهة برمجة التطبيقات

لا يعرض Explorer الآن سوى 12 مسار واجهة برمجة تطبيقات من نفس المصدر مطلوبة من واجهة المستخدم الخاصة به، بالإضافة إلى `GET /api/networkHealth`. تم إزالة ستة عشر تسجيل مسار قديم وCORS بديل عام. يتم فحص الطلبات مقابل واجهة برمجة تطبيقات دقيقة قبل البحث في Redis أو عمليات فحص جاهزية ADAMANT، مما يمنع إحياء نقاط النهاية المحذوفة عبر إدخالات ذاكرة تخزين مؤقت قديمة. تستخدم معاملات الاستعلام العامة الآن تحققًا صارمًا من الصحة وترقيم صفحات محدود. يطبق التطبيق حدًا ثابت النافذة داخل العملية بواقع 300 طلب واجهة برمجة تطبيقات في الدقيقة لكل عميل مع مراعاة الوكيل، وتخزين هوية محدود، ودلو فيضان يُغلق عند الفشل. الثقة في الوكيل العكسي صريحة ومُتحقق منها. تقلل رؤوس الأمان، وسياسة أمان المحتوى المقيدة، واستجابات الأخطاء المستقرة، ومهلات HTTP، وتسجيل الطلبات المقتصد للبيانات من سطح الهجوم المكشوف.

## التوافر وصحة الحالة

يُبلغ `GET /api/networkHealth` عن حالات `live` أو `degraded` أو `critical` أو `unavailable` المتماسكة، ويُعيد HTTP `503` فقط عندما لا يمكن إنتاج لقطة Node متماسكة. لم تعد أعطال Redis والخدمات الخارجية الاختيارية تسقط خدمة HTTP الأساسية وتقديم المحتوى الثابت. هوية ذاكرة التخزين المؤقت حساسة للكتل حيثما لزم الأمر، ومسار تحديث أسعار الصرف يحافظ على آخر قيم معروفة قابلة للاستخدام مع تجنب التحديثات المتداخلة. استقصاء Socket.IO متسلسل وواعٍ لدورة الحياة ومحدود أثناء حالات الفشل العلوي. تمنع تتبع الأجيال، وملكية المؤقتات الصريحة، وقمع الاستدعاءات القديمة، المساحات الأسماء المنفصلة أو المُعاد تشغيلها من مواصلة العمل القديم.

## البيانات غير الموثوقة وأمان المتصفح

تظل حمولات ADAMANT Node والأقران غير موثوقة حتى يتم تطبيعها أو التحقق منها. تُعرض قيم Network Monitor كنص، بينما تُقيَّد أهداف المسارات، والقيم المشتقة من CSS، والإحداثيات قبل الاستخدام. تشترك مسارات واجهة برمجة التطبيقات الأمامية والخلفية الآن في مصدر واحد للحقيقة لمنع انحراف العقد.

## التوافق وتأثير التكامل

واجهة برمجة تطبيقات Explorer المحتفظ بها هي تفصيل تنفيذي لواجهة المستخدم على الويب، وليست واجهة برمجة تطبيقات تكامل عامة. يجب أن تستخدم التطبيقات الخارجية [adamant-api-jsclient](https://github.com/Adamant-im/adamant-api-jsclient) للتكامل المباشر مع ADAMANT Node. يمكن للمشغلين استخدام `GET /api/networkHealth` لمراقبة Explorer. تظل مسارات الواجهة الأمامية والروابط العميقة الحالية متوافقة. يجب أن تُعد عمليات النشر خلف وكيل عكسي `trustedProxies` لتطابق البنية الفعلية. تُفضَّل عقد HTTPS؛ ولا يزال هناك احتياطي HTTP نصي قديم واحد للتوافق.

## النطاق والمتابعة

كانت هذه مراجعة كود وبنية المستودع لـ Explorer وحدود الثقة في وقت التشغيل الخاص به، وليست مراجعة بروتوكول تشفير أو إجماع سلسلة الكتل. مُحدد المعدل مقصود به لكل عملية، لذا يجب أن تفرض عمليات النشر متعددة النسخ حدًا إجماليًا عند الحافة أيضًا.
