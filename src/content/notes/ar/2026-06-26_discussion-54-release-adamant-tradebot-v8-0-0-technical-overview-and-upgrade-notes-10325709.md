## ملخص

تم إصدار ADAMANT tradebot v8.0.0 على الفرع `dev` مع وضع العلامة v8.0.0. هذه الترقية الكبرى للروبوت المفتوح المصدر (الأساسي) المخصص للسيولة تعيد محاذاة قاعدة الكود مفتوحة المصدر مع خط الهندسة الحالي للإصدار المميز (Premium)، مع الحفاظ على نطاق النسخة مفتوحة المصدر كما هو: أولوية REST، تركيز على التداول الفوري (spot)، وبدون إعادة إدخال وحدات خاصة بالإصدار المميز فقط. إذا كنت تشغّل الروبوت لرمز مدرج، فإن النتيجة العملية هي بيئة تشغيل أكثر موثوقية مع بدء تشغيل أنظف، ترقيات أكثر أمانًا على قواعد البيانات الحالية، وصلات تبادل محدثة، وواجهة ويب خاصة اختيارية (WebUI API) للعملاء الخارجيين.

## البنية

أصبح تسلسل التشغيل التمهيدي في `app.js` الآن صريحًا ومنظمًا. ينتظر الاتصال بقاعدة MongoDB وحالة `db.ready`، ثم ينفذ عمليات الهجرة المسجلة لقاعدة البيانات، يهيئ الخدمات مثل اتصال ADM عبر socket/الاستطلاع، وواجهة WebUI API الاختيارية، يحمّل بيانات الاتصال الأولية، ثم أخيرًا يبدأ وحدات التداول النشطة `mm_*`. هذا يقلل من حالات التزاحم عند البدء من الصفر ويجعل عمليات الترقية أكثر أمانًا.

تم تقسيم الملف الأحادي `modules/commandTxs.js` إلى معالجات متخصصة ضمن `modules/commands/` (account، orders، trade، info، features) إلى جانب وحدات البنية المشتركة. لا تزال الأوامر تصل عبر ADAMANT Messenger وTelegram عند التهيئة، لكن مسار الكود أصبح أسهل في الصيانة. تم استبدال وحدات استقبال ADM القديمة بـ `adamantApi.js` و`admTxChecker.js` و`admTxParser.js`، ما يُنسّق استقبال الأوامر مع `adamant-api` 3.x وأنماط الاتصال الحالية عبر socket/الاستطلاع.

تُنفَّذ هجرات قاعدة البيانات لمرة واحدة عند التشغيل عبر `modules/dbMigrations.js`. الهجرة الأولى المضمنة تُعيد تسمية الحقل القديم `type` في أوامر التداول إلى `side` بشكل آمن. يجب على النشرات الحالية نسخ قاعدة بيانات MongoDB احتياطيًا قبل أول تشغيل للإصدار v8، ثم التحقق من الأوامر المفتوحة والإحصائيات بعد الهجرة.

## واجهة ويب اختيارية (Bot API v1)

يأتي الإصدار v8.0 بواجهة HTTP خاصة تقع في الدليل `api/`. تستخدم هذه الواجهة خادم Fastify مع مصادقة JWT، والتحقق من صحة الطلبات عبر Zod، وSocket.IO لربط النقل. تشمل المسارات الأساسية ضمن `/api/v1` وظائف الإقلاع، السوق، الحساب، المعلمات، الأوامر، والرسائل. الواجهة اختيارية ومُطفأة افتراضيًا، وتم تصميمها للعملاء الرسوميين المُستضافين ذاتيًا، بينما يظل الروبوت هو العنصر الوحيد الذي يتواصل مع البورصات. يمكن تفعيلها عبر إعداد `private_webui` في ملف التكوين. يجب تعيين مفتاح سري قوي `private_webui_secret_key`، وتقييد `private_webui_allowed_ips`، ويجب عدم تعريض الواجهة للإنترنت العام ما لم تكن تعرف ما تفعله.

## وصلات التبادل

تشمل الوصلات المدعومة في النسخة الأساسية: Azbit، P2PB2B، StakeCube، Coinstore، FameEX (عبر وصلة FameEXnet الجديدة)، وNonKYC. يجب على المشغلين ملاحظة التغييرات الكبيرة: تتطلب واجهة API الخاصة بـ FameEX التبديل إلى وصلة FameEXnet في التكوين، كما تم إزالة XeggeX من النسخة مفتوحة المصدر. تم تحديث وحدات توفير السيولة جنبًا إلى جنب مع محولات المتداولين.

## التبعيات ودليل الترقية

من أبرز تحديثات التشغيل: `adamant-api` 3.x، `mongodb` 7، بالإضافة إلى إدخال `zod`، `fast-jwt`، و`json-parse-bigint`. تم إزالة Express من مسار واجهة ويب. للترقية من v7.x، أوقف تشغيل الروبوت، استرجع أحدث الكود، ثم ثبّت التبعيات:

```bash
pm2 stop tradebot
cd adamant-tradebot
git pull
npm i
```

تأكد من تثبيت Node.js 22.2 أو أحدث. دمج الحقول الجديدة من `config.default.jsonc` إلى ملف `config.jsonc` الخاص بك، تحقق من إعدادات وصلة FameEXnet، واحفظ نسخة احتياطية من قاعدة بيانات MongoDB لتمكين عمليات الهجرة عند التشغيل الأول. إذا كانت واجهة WebUI API مفعلة، راجع إعدادات الأمان الخاصة بها. أخيرًا، أعد تشغيل الروبوت باستخدام مدير العمليات الخاص بك. تظل التكوينات المُسماة تعمل كما هو متوقع.

## حدود نطاق النسخة مفتوحة المصدر (OSS)

يجلب الإصدار v8.0 جودة هندسة خط الإصدار المميز إلى شجرة النسخة مفتوحة المصدر دون إعادة إدخال افتراضات خاصة بالإصدار المميز. تم الحفاظ على الأساس الأولي للتداول الفوري (spot) مع أولوية REST، دون وصلات تبادل إلزامية عبر WebSocket، أو دعم للعقود الدائمة/المستقبلية، أو توسيع قائمة البورصات المدعومة.
