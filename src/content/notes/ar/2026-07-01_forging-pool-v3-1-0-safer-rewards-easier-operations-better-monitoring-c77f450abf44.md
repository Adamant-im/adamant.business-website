ADAMANT Forging Pool v3.1.0 هو تحديث موصى به لمُشغلَي المجمعات. يحسّن حساب المكافآت وموثوقية الدفع، ويقوّي منطق المحاسبة، ويحدّث بيئة التشغيل، ويسهّل العمليات اليومية.

### لماذا يُعد هذا الإصدار مهمًا

تتمثّل المسؤولية الأساسية لمجمع التوسيع في حساب مكافآت المصوتين بشكل صحيح ودفعها بأمان. ويتمحور إصدار v3.1.0 حول هذا بالتحديد. تم مراجعة تدفقات المكافآت والدفعات وتعزيزها لتقليل مخاطر الحالات الحدّية المتعلقة بإعادة المحاولة، أو الأعطال الجزئية، أو قيم المكافآت المحفوظة. يتم توحيد المكافآت المعلّقة قبل تسجيل الدفع، كما أصبحت حالة تقدّم المكافآت لكل مُصوّت مُتَبّعة بشكل أكثر أمانًا، بحيث لا يؤدي تعطّل النظام أو إعادة المحاولة إلى تكرار تحديثات المكافآت بالخطأ. ويجعل هذا من الإصدار مهمًا بشكل خاص للمُشغّلين المهتمين بصحّة المدفوعات وموثوقية التشغيل الطويلة.

### الجديد في v3.1.0

يتمثّل التغيير البنية التحتية الأكبر في الانتقال إلى تخزين مدعوم بـ MongoDB، مما يوفّر لمُشغّلي المجمعات طبقة تخزين أكثر متانة للبلوكات، والمُصوّتين، والمعاملات، وتاريخ المكافآت، والبيانات التشغيلية. بالنسبة للمجمعات الحالية، يتضمّن v3.1.0 أدوات للترقية من البيانات الأقدم المعتمدة على LowDB، بحيث يمكن للمشغلين التقدّم دون فقدان حالة المكافآت السابقة.

![Forging Pool v3.1.0: مكافآت أكثر أمانًا، عمليات أسهل، ومراقبة أفضل](/images/engineering-notes/medium/c77f450abf44/002-a7aa0fc331.webp)

كما يضيف هذا الإصدار توزيع مكافآت أكثر أمانًا ومحاسبة مدفوعات محسّنة، وتتبعًا آمنًا لإعادة المحاولة في حالة تقدّم المكافآت، وسجلات محسّنة للدفعات ومعالجة البلوكات، ونقطة نهاية `/api/health` لمراقبة خارجية، ودعمًا اختياريًا لكلمة مرور مندوب مشفرة، وأوامر `adm-pool` عبر واجهة سطر الأوامر (CLI) للعمليات encrypt وunlock وlock وstatus، وتصفية لوحة التحكم حسب العنوان أو الاسم، وعرضًا أوضح للمُصوّت/المندوب في الجداول، وتوثيقًا محدّثًا، وبيئة تشغيل افتراضية تعتمد على Node.js 22.13.0+.

### أمان أفضل للمشغلين

يمكن الآن لمُشغّلي المجمعات تشفير كلمة مرور المندوب باستخدام كلمة مرور المشغل. هذا خيار اختياري، وبالتالي تظل إعدادات كلمات المرور العادية الحالية مدعومة، لكن سير العمل الجديد يوفّر مسارًا أكثر أمانًا للإنتاج. مع كلمات المرور المشفرة، يمكن للمجمع البدء في حالة مقفولة. تظل مزامنة البلوكات ولوحة التحكم وواجهات برمجة التطبيقات العامة متاحة، بينما تظل المدفوعات وإشعارات ADM متوقفة حتى يقوم المشغل بفتح المجمع. وهذا يعني أن الخادم يمكنه الاستعادة أو إعادة التشغيل دون تعريض إمكانية الدفع فورًا.

### عمليات أسهل ومراقبة أفضل

يوفر واجهة سطر الأوامر الجديدة `adm-pool` أوامر بسيطة للمشغلين لأداء أهم إجراءات التشغيل الحساسة:

```sh
npm run adm-pool encrypt
npm run adm-pool unlock
npm run adm-pool lock
npm run adm-pool status
```

بدلًا من التعامل يدويًا مع كل تغيير حالة حساس داخل ملفات التهيئة أو سجلات العمليات، يحصل المشغلون على سير عمل تحكم مخصّص. توفّر نقطة النهاية الجديدة `/api/health` لقطة حالة خالية من الأسرار لأدوات المراقبة مثل Zabbix، أو لوحات التحكم المخصصة، أو فحوصات وقت التشغيل. بالاقتران مع تخزين MongoDB وسجلات أكثر وضوحًا، يصبح المجمع أسهل في المراقبة، وتصحيح الأخطاء، والصيانة على المدى الطويل.

### الترقية الموصى بها

يُوصى بترقية ADAMANT Forging Pool v3.1.0 لجميع مشغّلي المجمعات، خصوصًا أولئك الذين يشغلون مجمعات إنتاجية بمدفوعات منتظمة. قبل الترقية، يجب على المشغلين نسخ ملفات التهيئة وتاريخ المكافآت احتياطيًا، ومراجعة إعدادات MongoDB، واختبار عملية الترقية على نسخة من البيانات الحالية، والتحقق من إعدادات الدفع بعد الترقية.
