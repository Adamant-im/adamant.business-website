تُقدِّم هذه الإصدارة من ADAMANT خيار `cors` في ملف `config.json`. يمكن للمستخدمين الآن استخدام إما `inId` أو `isIn` في نقاط النهاية `/api/transactions` و`/api/chats/get`. على سبيل المثال، `/api/chats/get?InId=U6386412615727665758` و`/api/chats/get?isIn=U6386412615727665758` متكافئان.

أصبحت معرّفات المستخدمين في نقاط النهاية `/api/chats/get` و`/api/chatrooms` غير حساسة لحالة الأحرف الآن. على سبيل المثال، `/api/chatrooms/U6386412615727665758` و`/api/chatrooms/u6386412615727665758` متماثلان.

تقبل المعاملات و`POST /api/accounts/delegates` الآن المعاملة ككائن بسيط أو داخل الخاصية `transaction`.

```bash
curl -X POST https://endless.adamant.im/api/transactions/process
   -H 'Content-Type: application/json'
   -d '{"type": 0, "amount": 100000000, ...}'  # or {"transaction": { "type": 0, "amount": 100000000, ... }}
```

تضم نقطة النهاية `/api/states/get` الآن خاصية `confirmations`. تم تطبيق تحسين في استعلام `generatorPublicKey`. تم إصلاح خطأ "تم رفض الإذن للنظام public"، إلى جانب تغييرات طفيفة أخرى.

### التغييرات المُدخلة

لم تعد نقطة النهاية `/api/blocks` تُرجع الخاصية `count`.
