ADAMANT Forging Pool v3.1.0 هو تحديث موصى به لمشغلي المجمعات. يُصلح هذا الإصدار ويُعزز حسابات المكافآت ومعالجة الدفعات المجدولة، ويُدخل عدة تغييرات هيكلية يجدر مراجعتها قبل ترقية المجمعات المنتجة.

## سبب الترقية

تستهدف أبرز التغييرات سلامة الدفعات، وأمن المشغل، وسهولة الصيانة على المدى الطويل. أصبحت المكافآت المعلقة تُعادل الآن قبل حساب الدفعات، وتُحفظ تقدم المكافآت لكل ناخب بالنسبة للبلوكات المُسَجَّلة. أصبحت مسارات إعادة المحاولة والانهيار أكثر أمانًا ضد تحديثات المكافآت المكررة. أصبحت مسارات الدفعات، وتحليل البلوكات، وسلوك التخزين تمتلك الآن اختبارات أكثر تركيزًا، كما أصبحت السجلات والإشعارات أكثر وضوحًا في حالات فشل التشغيل.

## التخزين والترحيل

يتم الآن دعم تخزين المجمع عبر MongoDB. يتضمن هذا الإصدار أدوات ترحيل لبيانات المجمعات القديمة المستندة إلى LowDB، تقع في `scripts/migrate-lowdb-mongodb/`، إلى جانب اختبارات الترحيل، وإعداد فهارس MongoDB، ومراجع تكوين محدّث، وإرشادات ترحيل في ملف README. يجب على المشغلين نسخ بيانات المجمع الحالية احتياطيًا، واختبار الترحيل على نسخة من البيانات، والتحقق من إجمالي المكافآت المعلقة والمُستلمة قبل تحويل الدفعات المنتجة إلى المجمع المحدّث.

## أمن المشغل

يُضيف الإصدار v3.1.0 دعمًا اختياريًا لكلمة مرور مشفرة. تظل تهيئة كلمات المرور النصية الحالية مدعومة، لكن يمكن للمشغلين الآن تشفير كلمة مرور المفوّض وفتح المجمع الجاري فقط عند تفعيل الدفعات:

```sh
npm run adm-pool encrypt
npm run adm-pool unlock
npm run adm-pool lock
npm run adm-pool status
```

تستخدم واجهة التحكم مقبس Unix محليًا مخصصًا للمالك فقط. مع كلمة المرور المشفرة، يمكن تشغيل المجمع مغلقًا: تظل مزامنة البلوكات، ولوحة القيادة، والواجهة البرمجية العامة متاحة، بينما تبقى الدفعات وإشعارات ADM متوقفة حتى يتم الفتح.

## المراقبة ولوحة القيادة

يُضيف هذا الإصدار نقطة نهاية `/api/health` لمراقبة خارجية دون الحاجة إلى أسرار، إلى جانب حالة الدفعات المغلقة في لوحة القيادة. أصبحت تصفية الناخبين والمعاملات حسب العنوان أو الاسم مدعومة الآن، وتُعرض أسماء المفوّض والناخب أسفل العناوين عند توفرها. تُكمَل التغييرات بإصلاحات الفرز العددي، وتحسين روابط المستكشف، وتحسينات تخطيط لوحة القيادة.

## ملاحظات التشغيل والترقية

أصبح الحد الأدنى لبيئة التشغيل الآن Node.js 22.13.0+ وnpm 10+. تم تحديث ملفات README، وCONTRIBUTING، وإرشادات المستودع لإعداد، وترحيل، ومراقبة، وسلامة الأسرار، وتدفق المساهمة.

قبل الترقية، قم بعمل نسخة احتياطية من التهيئة وتاريخ المكافآت، وراجع إعدادات MongoDB في `config.default.jsonc`، واختبر ترحيل LowDB إلى MongoDB على نسخة من البيانات. بعد الترحيل، تحقق من إجمالي المكافآت المعلقة والمُستلمة، وراجع إعدادات محفظة الدفعات والصيانة، وقم ببناء واختبار لوحة القيادة بشكل أولي، وتحقق من `/api/health`. لا تُفعّل الدفعات إلا بعد التحقق من سجلات التشغيل وحالة المجمع.
