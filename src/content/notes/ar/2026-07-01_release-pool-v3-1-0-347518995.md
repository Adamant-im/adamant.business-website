ADAMANT Forging Pool v3.1.0 هو تحديث موصى به لمُشغلِي المجمعات.

يُصلح هذا الإصدار مسارات حساب المكافآت والتسوية المحاسبية للدفعات، وينقل تخزين المجمع إلى مجموعات مدعومة بـ MongoDB، ويضيف أدوات للترحيل من LowDB إلى MongoDB للبيانات القديمة. ويُدخِل تتبعًا آمنًا ضد إعادة المحاولة لحالة المكافآت لكل ناخب لمنع تكرار المكافآت بعد حدوث تعطل أو تشغيل جزئي، ويعمل على توحيد المكافآت المعلقة المخزنة قبل التسوية المحاسبية للدفعات، ويضيف دعمًا اختياريًا لعبارة المرور المشفرة للمجمع. يقدّم واجهة سطر الأوامر الجديدة `adm-pool` أوامر `encrypt` و`unlock` و`lock` و`status`، إلى جانب مقبس تحكم محلي للمسؤول فقط لتنفيذ أوامر القفل/الإلغاء أثناء التشغيل. كما يتيح مسار النهاية `/api/health` المراقبة الخارجية دون الحاجة إلى أسرار. أصبح لوحة التحكم الآن تشمل تصفية الناخبين والمعاملات، وأسماء المفوّضين/الناخبين، وتصحيحات لفرز القيم الرقمية، وروابط محسّنة. تم تحديث التوثيق لإعداد المجمع، والتكوين، والترحيل، والمراقبة، وسير عمل المشغلين. تم تحديث الحد الأدنى لبيئة التشغيل إلى Node.js 22.13.0+ وnpm 10+.

يُعد هذا التحديث موصى به لأنه يتضمن إصلاحات وضوابط لحسابات المكافآت والدفعات المجدولة. قبل الترقية، يجب على المشغلين نسخ ملف تهيئة المجمع وسجل المكافآت احتياطيًا، ومراجعة تهيئة MongoDB في `config.default.jsonc`، ومراجعة دليل الترحيل في `README.md`، واختبار عملية الترحيل على نسخة من البيانات الحالية، والتحقق من إعدادات الدفعات، وإعدادات التبرعات، وإعدادات محفظة الصيانة بعد الترحيل. عبارات المرور المشفرة اختيارية؛ وتظل تهيئة عبارات المرور العادية الحالية مدعومة. يمكن للمشغلين تشفير عبارة مرور المفوّض باستخدام:

```sh
npm run adm-pool encrypt
```

ثم إلغاء قفل المجمع العامل باستخدام:

```sh
npm run adm-pool unlock
```
