لضمان وضوح واتساق القضايا عبر منظمة ADAMANT، استخدم بادئات قصيرة في العنوان عند إنشاء قضية جديدة. يساعد هذا الجميع على فهم نوع المهمة بسرعة.

## البادئات الشائعة

استخدم **[Bug]** للإبلاغ عن الأخطاء أو الأعطال أو السلوك غير المتوقع. استخدم **[Feat]** للوظائف الجديدة و**[Enhancement]** للتحسينات على الميزات الحالية. استخدم **[Refactor]** لإعادة هيكلة الكود دون تغيير الوظائف. استخدم **[Docs]** للتوثيق أو الإرشادات أو تحديثات ملفات README. استخدم **[Test]** لإضافة اختبارات أو تحسينها. استخدم **[Chore]** للمهام الروتينية مثل تحديث التبعيات أو تعديلات CI/CD.

## بادئات محددة بالمشروع

استخدم **[Task]** لمهمة عامة قد لا تكون مرتبطة مباشرة بالبرمجة. استخدم **[Composite]** لمهمة تتضمن عدة مهام فرعية. استخدم **[UX/UI]** لتغييرات واجهة المستخدم أو تجربة المستخدم.

## الأفكار والمقترحات

استخدم **[Proposal]** للمقترحات الرسمية، و**[Idea]** للأفكار التي يجب أخذها بعين الاعتبار أو مناقشتها أكثر، و**[Discussion]** للمواضيع التي قد تتطلب مدخلات أوسع. ومع ذلك، فإن أفضل مكان لهذه الأنواع الثلاثة هو [منتدى ADAMANT](https://github.com/orgs/Adamant-im/discussions) وليس القضايا.

## أمثلة

```
[Bug] Crash when syncing wallet on iOS
[Feature] Add support for biometric auth
[Docs] Update installation guide for Ubuntu 24
[Refactor] Split TradeBot config module
[Composite] Add decentralized file transfer across ADM messenger apps
[UI/UX] Improve Home screen design
[Task] Release iOS app v3.10.1 to the App Store
```

استخدم بادئتين كحد أقصى لتجنب إثقال العنوان.
