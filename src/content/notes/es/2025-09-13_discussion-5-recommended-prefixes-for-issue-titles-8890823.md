Para mantener las incidencias claras y consistentes en toda la organización ADAMANT, use prefijos cortos en el título al crear una nueva incidencia. Esto ayuda a que todos entiendan rápidamente el tipo de tarea de un vistazo.

## Prefijos comunes

Use **[Bug]** para errores, bloqueos o comportamientos inesperados. Use **[Feat]** para funcionalidades nuevas y **[Enhancement]** para mejoras en funciones existentes. Use **[Refactor]** para refactorización de código que no cambia la funcionalidad. Use **[Docs]** para documentación, guías o actualizaciones de README. Use **[Test]** para añadir o mejorar pruebas. Use **[Chore]** para tareas rutinarias como actualizaciones de dependencias o ajustes en CI/CD.

## Prefijos específicos del proyecto

Use **[Task]** para una tarea general que puede no estar directamente relacionada con programación. Use **[Composite]** para una tarea que incluye múltiples subtareas. Use **[UX/UI]** para cambios en la interfaz de usuario o experiencia de usuario.

## Ideas y propuestas

Use **[Proposal]** para propuestas formales, **[Idea]** para ideas que se deben considerar o discutir más a fondo, y **[Discussion]** para temas que puedan requerir aportes más amplios. Sin embargo, el mejor lugar para estos tres es el [foro de ADAMANT](https://github.com/orgs/Adamant-im/discussions) en lugar de las incidencias.

## Ejemplos

```
[Bug] Crash when syncing wallet on iOS
[Feature] Add support for biometric auth
[Docs] Update installation guide for Ubuntu 24
[Refactor] Split TradeBot config module
[Composite] Add decentralized file transfer across ADM messenger apps
[UI/UX] Improve Home screen design
[Task] Release iOS app v3.10.1 to the App Store
```

Use como máximo uno o dos prefijos para evitar sobrecargar el título.
