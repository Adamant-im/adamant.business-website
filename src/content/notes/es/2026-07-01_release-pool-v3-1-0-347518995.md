ADAMANT Forging Pool v3.1.0 es una actualización recomendada para operadores de pools.

Esta versión corrige y refuerza las rutas de cálculo de recompensas y contabilidad de pagos, mueve el almacenamiento del pool a colecciones respaldadas por MongoDB y añade herramientas para la migración de LowDB a MongoDB para datos antiguos del pool. Introduce un seguimiento de progreso de recompensas por votante seguro ante reintentos para evitar recompensas duplicadas tras fallos o ejecuciones parciales, normaliza las recompensas pendientes almacenadas antes de la contabilidad de pagos, y añade soporte opcional para frases de acceso cifradas del pool. La nueva CLI `adm-pool` para operadores incluye los comandos `encrypt`, `unlock`, `lock` y `status`, junto con un socket de control local solo para el propietario que permite comandos de bloqueo/desbloqueo en tiempo de ejecución. Un punto final `/api/health` permite el monitoreo externo sin necesidad de secretos. El panel ahora incluye filtros para votantes y transacciones, nombres de delegados/votantes, correcciones para orden numérico y enlaces mejorados. La documentación se ha actualizado para configuración, migración, monitoreo y flujos de trabajo de operadores. El entorno de ejecución ahora requiere Node.js 22.13.0+ y npm 10+.

Esta actualización es recomendada porque incluye correcciones y protecciones para el cálculo de recompensas y pagos programados. Antes de actualizar, los operadores deben hacer una copia de seguridad de la configuración actual del pool y del historial de recompensas, revisar la configuración de MongoDB en `config.default.jsonc`, consultar la guía de migración en `README.md`, probar el flujo de migración en una copia de los datos existentes y verificar la configuración de pagos, donaciones y billetera de mantenimiento tras la migración. Las frases de acceso cifradas son opcionales; las configuraciones con frases de acceso en texto claro siguen siendo compatibles. Los operadores pueden cifrar la frase de acceso del delegado con:

```sh
npm run adm-pool encrypt
```

Luego desbloquear un pool en ejecución con:

```sh
npm run adm-pool unlock
```
