Esta versión introduce una compilación universal para macOS y migra el proyecto a una nueva configuración de ESLint. Se ha añadido un archivo de notas de versión al repositorio. El sistema de verificación de estado se ha mejorado: ahora se muestra un estado de "Actualizando…" antes de los intentos de solicitud HTTP, las verificaciones iniciales y posteriores siguen un algoritmo de solicitud refinado, y el sistema comprueba `alt_ip` cuando un dominio web no es accesible. Se ha corregido la recuperación de la verificación de estado en segundo plano tras la suspensión del sistema, y se ha resuelto el informe incorrecto del estado del nodo. Se ha actualizado la pantalla de billeteras, junto con el script para adamant-wallets, y se ha creado una pantalla de prueba para dichas herramientas.

Ahora los mensajes pueden agruparse en grupos con un diseño de burbujas. Se han aplicado varias actualizaciones de estilo en toda la aplicación, incluyendo estilos de entrada de mensajes, diseño del formulario de exportación de claves y mejoras generales de coherencia visual. El chat de bienvenida a ADAMANT ahora incluye un campo de entrada deshabilitado. Se han añadido comprobaciones de Playwright solo para desarrollo como parte de un esfuerzo más amplio para unificar estilos y modernizar la interfaz de usuario y la experiencia de usuario.

Se incluyen numerosas correcciones de errores. Las fechas en los chats ahora se actualizan correctamente. Ya no aparecen indicadores de sincronización falsos al abrir la pantalla de nodos. Se han corregido las cantidades incorrectas de transacciones para BTC, DOGE y DASH al enviar a la propia dirección. Se ha corregido el marcado que fallaba al responder a un mensaje que contenía un enlace largo. Se ha resuelto un error de referencia nula relacionado con `maintainScrollPosition`. Los chats estáticos ya no son de solo lectura tras volver a iniciar sesión. Las comisiones de transacción para las monedas ERC20 ahora son consistentes entre PWA e iOS. Se han corregido la navegación de transacciones y regresiones de interfaz localizada en las pantallas de la PWA. Los símbolos de billetera heredados ahora se normalizan para evitar un fallo en la ruta `/options`. Se ha corregido el nombre del APK publicado, y se han resuelto problemas de código fuente detectados por ESLint 9. Se ha corregido actividad de commit incorrecta en el repositorio.

Las dependencias y herramientas generales se han actualizado eliminando advertencias de compilación. Node.js se ha actualizado de la versión 20 a la 22. Se ha implementado un Release Drafter organizacional y un flujo de trabajo reutilizable. Se ha añadido un flujo de trabajo de GitHub Actions y un hook git postinstall de Husky para ESLint. Se ha añadido documentación con pautas de operación para agentes de IA para la PWA, y se han actualizado el README y los metadatos de versión.

### Cambios importantes

El soporte para Klayr (KLY) se ha eliminado por completo de la billetera. Los usuarios que posean activos KLY deben saber que esta criptomoneda ya no es compatible con la versión 4.11.0 ni versiones posteriores.
