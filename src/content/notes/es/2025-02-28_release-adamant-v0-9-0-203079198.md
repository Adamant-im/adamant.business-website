Esta versión introduce varias mejoras en la API y correcciones para el software del nodo ADAMANT. El punto final `/api/delegates/get` ahora acepta un parámetro `address` además de los parámetros previamente soportados `username` y `publicKey`. El uso es sencillo:

```bash
/api/delegates/get?address=U1234567890
```

El soporte para WebSocket se ha mejorado para permitir la suscripción a múltiples tipos de transacciones en lugar de solo uno. Para las transacciones de mensajes, un nuevo evento `assetChatTypes` permite a los clientes suscribirse a tipos específicos de mensajes.

El punto final `/api/node/status` ahora devuelve propiedades adicionales al combinar datos de `/api/loader/status` y `/api/loader/status/sync`, reduciendo así la cantidad de solicitudes necesarias para obtener información completa sobre el estado de la blockchain:

```json5
{
  // ...
  "loader": {
    "loaded": true,
    "now": 45345521,
    "syncing": false,
    "consensus": 100,
    "blocks": 0,
    "blocksCount": 0
  }
}
```

Se han corregido varios errores. Se resolvió un problema de conexión con pares cuyos nombres de versión incluyen metadatos de prelanzamiento o compilación (como `+alpha` o `-dev.0`), logrando cumplimiento total con SemVer 2.0. Se corrigió la eliminación de pares tras errores de conexión únicos, y se arregló la validación del archivo de configuración.

El punto final `/delegates/get` se ha optimizado para búsquedas de una sola transacción. Se actualizaron dependencias, se realizaron pequeñas mejoras en los registros (logs) y se añadió una nueva guía para colaboradores respecto a las pruebas.
