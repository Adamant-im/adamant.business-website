Esta versión mejora las conexiones socket al devolver el campo `recipientPublicKey`. La API de Transacciones, incluyendo KVS y Chats, ahora devuelve `block_timestamp` junto con los datos de respuesta existentes.

Se ha aplicado una corrección para las migraciones, resolviendo problemas que anteriormente afectaban las rutas de actualización de la base de datos. El punto de conexión `/states/get/` se ha ampliado para admitir los parámetros `SenderIds` y `keyIds`, y ahora también acepta solicitudes POST, ampliando las formas en que los clientes pueden consultar información de estado.

La documentación se ha actualizado para reflejar estos cambios.

### Cambios importantes

No se introducen cambios importantes en esta versión.
