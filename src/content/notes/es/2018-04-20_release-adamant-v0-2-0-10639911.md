Esta versión introduce el sistema de Almacenamiento por Clave Valor (KVS), también denominado estados, que es necesario para implementar funciones adicionales requeridas en las aplicaciones cliente. Las tarifas por operaciones KVS son las mismas que para los mensajes. Se han añadido nuevos puntos de acceso API para soportar estas funciones, específicamente el punto de acceso `/states/` con las acciones `store` y `get`.

Los nuevos puntos de acceso no soportan cifrado en el nodo, por lo que toda la firma debe realizarse en el cliente. Un ejemplo de uso está disponible en el paquete adamant-console.

### Cambios importantes

Esta versión incluye cambios importantes en la API, razón por la cual se incrementó la versión menor. La incorporación del sistema KVS y sus puntos de acceso asociados altera la superficie API existente, lo que obliga a las aplicaciones cliente a adaptarse a los nuevos puntos de acceso y a los requisitos de firma en el cliente.
