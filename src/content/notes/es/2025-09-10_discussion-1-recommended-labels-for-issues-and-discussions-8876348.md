ADAMANT utiliza un conjunto unificado de etiquetas en ambos, Problemas y Discusiones. Aunque algunas etiquetas pueden no aplicar por igual a ambos, compartir el conjunto completo brinda flexibilidad a la comunidad. Las etiquetas predeterminadas de GitHub se mantienen en minúsculas, mientras que las etiquetas personalizadas comienzan con mayúscula (excluyendo prefijos).

Las etiquetas integradas de GitHub se conservan tal como están: `bug`, `documentation`, `duplicate`, `enhancement`, `good first issue`, `help wanted`, `invalid`, `question`, y `wontfix`.

## Estados especiales

Tres etiquetas de estado personalizadas indican prioridad y alcance. `High priority` marca problemas urgentes que requieren atención inmediata. `Primary feature` identifica funciones esenciales para la funcionalidad del producto. `Composite task` señala elementos de trabajo grandes o complejos que se descomponen en múltiples subtareas.

## Lenguajes, frameworks y tecnologías

Las etiquetas de tecnología rastrean la pila relevante: `JavaScript` y `TypeScript` para código frontend y backend, `Vue` para características de Vue.js, `CSS` para estilos y temas, `NodeJS` para lógica backend y APIs, `Swift` para iOS, `Kotlin` para Android, `IPFS` para almacenamiento descentralizado, `Electron` para empaquetado de escritorio, y `Capacitor` para compilaciones móviles e integraciones.

## Producto, plataforma y flujo de trabajo

Las etiquetas específicas de plataforma incluyen `Web`, `Mobile` y `Desktop` para problemas específicos del cliente, mientras que `UX/UI` cubre mejoras de interfaz y experiencia. `Performance` se enfoca en optimizaciones de velocidad, almacenamiento o red. `Protocol & AIPs` abarca cambios en el protocolo de la blockchain ADM e interacciones entre nodos, e `Integration` maneja conexiones con blockchains externas, servicios o APIs. `Auth` incluye frases de acceso, claves, passkeys, biométricos y almacenamiento local seguro. `Nodes` rastrea el software, APIs, conectividad y consenso de nodos ADM. `DB` cubre PostgreSQL, MongoDB, migraciones y consultas. `Infrastructure` abarca nodos de monedas y de ADAMANT, servidores, VPS, Ansible, Nginx, Docker y Zabbix. `APIs` trata sobre diseño de APIs y endpoints. `Automation`, `CI/CD` y `Linter` cubren scripts, GitHub Actions, pipelines y verificaciones de calidad de código (ESLint/Stylelint).

## Etiquetas para discusiones

Las etiquetas específicas de discusión ayudan a categorizar conversaciones. `Guideline` cubre mejores prácticas, estándares de codificación y reglas de contribución. `Blockchain` y `Cryptocurrency` abordan mecánicas del libro mayor y discusiones sobre ADM u otros tokens. `Messaging` se relaciona con mensajeros, protocolos de comunicación y menciones. `Decentralization` y `Privacy` rastrean esfuerzos para reducir dependencias centralizadas y proteger el anonimato del usuario. `Encryption` y `Security` cubren cifrado de extremo a extremo, algoritmos criptográficos, gestión de claves y vulnerabilidades. `Governance` maneja AIPs y propuestas. `Community`, `Ideas` y `Research` apoyan retroalimentación, sugerencias para la hoja de ruta a largo plazo e investigaciones. `Trading` cubre bots, mercados, exchanges y liquidez. `Memes` es para contenido divertido de la comunidad. `Marketing` y `Marketing offers` abordan promoción, crecimiento comunitario, asociaciones, listados en exchanges y participación con KOL.
