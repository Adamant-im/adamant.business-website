Esta versión añade soporte para un `returnAsset` argumento en los endpoints `/api/transactions` y `/api/transactions/get`. Cuando se proporciona, este argumento fuerza a la API a incluir el activo de la transacción en la carga de respuesta. Este cambio sienta las bases para consolidar los endpoints de la API en futuras versiones, reduciendo la cantidad de llamadas necesarias para obtener los detalles completos de una transacción.
