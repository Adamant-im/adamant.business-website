このリリースでは、クライアントアプリケーションに必要な追加機能を実装するためのキー値ストレージ（KVS）システム（状態、states とも呼ばれる）を導入します。KVS 操作の手数料はメッセージと同額です。これらの機能をサポートするために、`/states/` エンドポイントに `store` および `get` アクションが新たに追加されました。

新しいエンドポイントはノード側の暗号化をサポートしていないため、すべての署名はクライアント側で行う必要があります。使用例は adamant-console パッケージで確認できます。

### 互換性のない変更点

このリリースには API の互換性を損なう変更が含まれているため、マイナーバージョンをインクリメントしています。KVS システムおよび関連エンドポイントの追加により、既存の API フェースが変更され、クライアントアプリケーションは新しいエンドポイントおよびクライアント側署名の要件に適応する必要があります。
