このリリースでは、データベース移行スクリプトのログ出力が追加されました。ADAMANTノードは、移行スクリプトの実行前と実行後に以下のメッセージをログに出力するようになります：

```
[inf] 2023-12-22 04:00:25 | Found 5 pending migrations. Start executing, this may take a while…
[inf] 2023-12-22 04:17:13 | Migrations have been successfully completed.
```

`cors.origin`プロパティのデフォルト値が、リクエスト元を反映する（`true`）設定から`*`に変更されました。ADAMANTノードのCORS設定の詳細については、ドキュメントを参照してください。

```json5
{
  "cors": {
    "origin": "*" // default
  }
}
```

`config.json`ファイルが`.gitignore`に追加されました。これにより、ユーザーはADAMANTノードを起動する前に、`config.json`ファイルを手動で作成するか、`config.default.json`をコピーして作成する必要があります。

```
cp config.default.json config.json
```

### Breaking changes

`Access-Control-Allow-Origin`のデフォルトCORSヘッダー値が、リクエスト元の反映から`*`に変更されました。以前のオリジン反映動作に依存しているユーザーは、`config.json`を適切に更新する必要があります。また、`config.json`はgitによって無視されるようになったため、ノード起動前にこのファイルを手動で作成またはコピーする必要があります。
