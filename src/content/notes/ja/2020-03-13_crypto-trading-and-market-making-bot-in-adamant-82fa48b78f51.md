ADAMANT 取引ボットは、さまざまな暗号通貨取引所をサポートしています。手動または自動取引に利用でき、現時点では投機的な利益を狙うよりも、注文板の充実と取引高の創出を主な目的としています。

### 取引ボットに対する現実的な期待

取引ボットは利益を保証するものではありません。成功する確率はほぼ五分五分であり、ボットが確実に利益を生むと主張する人は正直ではありません。企業が取引ボットを販売しているのは、ボットを使用するか手動で取引するかに関わらず、暗号通貨取引には大きなリスクが伴うためです。ADAMANT ボットの利益追求型の取引機能は現時点では限定的であり、主な利用用途はマーケットメイキングです。

### マーケットメイキングが重要な理由

暗号通貨取引所における取引高の多くは人工的です。小規模な取引所に上場している小規模なトークンプロジェクトは、人気のあるコインでさえも取引高を獲得するのが難しいため、取引高がゼロになることがよくあります。目に見える取引高がなければ、ユーザーは購入をためらいますし、他の取引所も上場を拒否するでしょう。そのため、プロジェクト運営者は自ら取引高を創出し、注文板を埋める必要があり、その過程で取引所の手数料を支払わざるを得ません。

### ボットの仕組み

取引ボットは永続的に動作するサーバーサイドのプログラムです。インストール後、対象の取引所と取引ペアを設定します。ボットは受信したコマンドを監視し、設定された戦略に従って取引を実行し、すべての操作について通知を送信します。コマンドは ADAMANT Messenger 経由で発行されるため、管理者としての自身のアカウントとボット用のアカウント、合計2つの ADM アカウントが必要です。

### 導入のための要件

Linux と Node.js の基礎知識に加え、任意のクラウドホスティングプロバイダーから最小限の仮想サーバーを用意する必要があります。フルノードの ADAMANT ノードを実行する必要はありません。2つの ADAMANT アカウントを作成する必要があります。1つは管理者用の個人アカウントで、そのアドレスを `admin_accounts` 設定フィールドに入力します。もう1つはボット用のアカウントで、その `passPhrase` を設定ファイルに記載します。ボットに送信する各メッセージには 0.001 ADM が消費されますが、ADAMANT のバウンティプログラムを通じて、数年分に十分な量の無料 ADM トークンを取得できます。

取引所側では、取引所の API 設定画面で作成した API キーが必要です。取引所上で取引ペアの残高を入金し、ボット起動前に選択したペアの注文板に少なくとも1つの買い注文（bid）と売り注文（ask）があることを確認してください。最後に、ボットをインストールし、設定ファイルを調整して実行します。

### コマンド

ボットは ADAMANT Messenger 経由でコマンドを受け付けます。利用可能なコマンドを確認するには `/help` を使用し、詳細は完全なコマンドリファレンスを参照してください。
