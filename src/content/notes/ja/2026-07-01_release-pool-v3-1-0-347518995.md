ADAMANT Forging Pool v3.1.0は、プール運営者向けの推奨アップデートです。

このリリースでは、報酬計算および支払い会計のパスを修正および強化し、プールのストレージをMongoDB対応コレクションに移行し、旧バージョンのプールデータ向けにLowDBからMongoDBへの移行ツールを追加しています。クラッシュや部分実行後に重複報酬を防ぐため、投票者ごとのリトライ安全な報酬進捗追跡を導入し、支払い会計前の保留報酬の正規化を実施しています。また、オプションの暗号化されたプールパスフレーズのサポートも追加されました。新しい `adm-pool` 運営者CLIは、`encrypt`、`unlock`、`lock`、`status` コマンドを提供し、ランタイムでのロック/アンロック操作のためのローカルかつ所有者のみアクセス可能なコントロールソケットも備えています。`/api/health` エンドポイントにより、シークレット不要の外部監視が可能になりました。ダッシュボードには、投票者およびトランザクションのフィルタリング、デリゲート/投票者の名前表示、数値ソートの修正、リンクの改善が加えられています。セットアップ、設定、移行、監視、運営ワークフローに関するドキュメントも更新されています。ランタイムのベースラインはNode.js 22.13.0+およびnpm 10+に更新されています。

これは、報酬計算およびスケジュールされた支払いに関する修正と保護機能を含むため、推奨されるアップグレードです。アップグレード前に、運営者は既存のプール設定および報酬履歴をバックアップし、`config.default.jsonc` 内のMongoDB設定を確認し、`README.md` 内の移行ガイドを確認し、既存データのコピーで移行手順をテストし、移行後に支払い設定、寄付設定、メンテナンスウォレット設定を確認する必要があります。暗号化されたパスフレーズの使用は任意です。既存のプレーンテキストのパスフレーズ設定は引き続きサポートされます。運営者は以下のコマンドでデリゲートのパスフレーズを暗号化できます：

```sh
npm run adm-pool encrypt
```

その後、実行中のプールは以下でアンロックできます：

```sh
npm run adm-pool unlock
```
