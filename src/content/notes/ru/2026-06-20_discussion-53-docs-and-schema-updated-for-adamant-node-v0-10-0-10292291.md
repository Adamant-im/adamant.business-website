Вместе с **ADAMANT Node v0.10.0** стек для разработчиков был обновлён, чтобы оставаться синхронизированным с нодой: обновлены спецификация API (`adamant-schema`) и документация (`adamant-docs`), а также добавлены локальная сеть и недавно перезапущенная тестовая сеть. Ниже краткое резюме для всех, кто создаёт решения на базе ADAMANT.

## adamant-schema (спецификация API)

Спецификация обновлена с **OpenAPI 3.0.3 → 3.2.0**, её версия приведена в соответствие с `0.10.0` и проверена на работающей ноде. Транзакции теперь поддерживают **`timestampMs`**, обеспечивая временную метку с точностью до миллисекунд в дополнение к существующему `timestamp` с точностью до секунд. Ответы о статусе ноды стали более информативными: добавлены **`nodeTimestampMs`**, **`unixTimestampMs`** и объект **`loader`**, который предоставляет данные о `syncing`, `consensus`, `blocks` и `blocksCount`.

Новый эндпоинт **`GET /peers/get`** позволяет искать пир по IP и порту. Новые параметры запроса **`returnUnconfirmed`** и **`includeDirectTransfers`** заменяют устаревший `withoutDirectTransfers`. В список серверов добавлены тестовые ноды.

Интерактивный Swagger UI на [schema.adamant.im](https://schema.adamant.im) теперь поддерживает поиск операций в реальном времени, проверку работоспособности отдельных нод с метками версии API и автоматический выбор здоровой ноды основной сети. Инструментарий обновлён до Node.js 22, TypeScript и Express 5, а также сгенерирован новый типизированный клиентский путь для потребителей.

## adamant-docs (документация)

Справочник по API обновлён до версии **v0.10.0** и снабжён тегами версий в боковой панели, чтобы документация и сеть всегда оставались синхронизированными. Добавлены новые страницы по темам **консенсуса и валидации транзакций**, **синхронизации** и эндпоинта **loader / статус ноды**. Полностью задокументированы семантика **`timestampMs`** и обновлена документация по API **peers**.

Руководства **run-your-own-node** расширены и теперь охватывают установку (включая macOS), настройку, автозапуск, загрузку (bootstrap) и восстановление ноды. Появились отдельные страницы для запуска **localnet** и подключения к **testnet**.

## Localnet и testnet

**Localnet** позволяет развернуть полную сеть ADAMANT локально для разработки и тестирования без взаимодействия с публичной инфраструктурой. **Testnet** был перезапущен и приведён в соответствие с v0.10.0, что даёт возможность проверить интеграции в реальных сетевых условиях до выхода в основную сеть. Вместе эти компоненты обеспечивают плавный и полностью документированный путь **local → testnet → mainnet**.
