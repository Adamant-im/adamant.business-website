Сеть ADAMANT работает на узлах, управляемых сообществом, — независимых серверах, которые пересылают зашифрованные сообщения, предоставляют API для мессенджеров и обеспечивают честность блокчейна. ADAMANT Node v0.10.0 — это значительное обновление, которое делает эту инфраструктуру быстрее в эксплуатации, проще в отладке и лучше соответствующей реальному поведению современных клиентов ADAMANT. ADAMANT не гонится за показателями пропускной способности; вместо этого он создаёт децентрализованный слой доверия для коммуникаций, на который могут полагаться мессенджеры, кошельки и форки без ущерба для приватности.

### Улучшенные API для реальных мессенджеров

Клиентам нужны точная сортировка транзакций, временные метки в миллисекундах и возможность отображать не подтверждённые сообщения, пока чат ещё распространяется по сети. Версия v0.10.0 добавляет поле `timestampMs` для упорядочивания транзакций с точностью до долей секунды, не нарушая при этом существующее поле `timestamp`. Эндпоинты списков теперь принимают параметр `?returnUnconfirmed=1`, чтобы включать транзакции из mempool, когда это уместно. Новый параметр `includeDirectTransfers` заменяет устаревшую логику `withoutDirectTransfers`, упрощая фильтрацию чатов. Поле `count` теперь возвращается как число, а не как строка, что упрощает парсинг на стороне клиента во всех эндпоинтах списков.

### Более быстрая и устойчивая сеть

Теперь узлы могут поддерживать WebSocket-соединения между пиринговыми узлами, а не только HTTP. Это снижает задержку при передаче блоков и транзакций и даёт операторам больше гибкости в участии узла в сетевой меш-структуре. В сочетании с улучшенной логикой синхронизации и переписанным пулом транзакций узел более предсказуемо работает в условиях высокой нагрузки.

### Инструменты для операторов, которые ценят ваше время

Запуск узла не должен требовать глубокого знания устаревшего JavaScript. Версия v0.10.0 включает модернизированные скрипты установки и восстановления для Ubuntu/Debian и CentOS/RHEL, вспомогательные утилиты localnet для разработчиков, переопределение конфигурации для поэтапного развёртывания и структурированное логирование с ротацией, чтобы операторы могли эффективно диагностировать проблемы. В релизе также документируются рекомендации по корректному завершению работы — принудительное `kill -9` на работающем узле может повредить зеркала состояния в памяти, поэтому в v0.10.0 в документации операторов явно указан правильный порядок остановки.

### Безопасность без излишней драмы

Этот выпуск переносит криптографию на привязки `sodium-native` и усиливает проверку транзакций в P2P-сети. Проверки временных меток, которые ранее защищали Public API, теперь также применяются, когда транзакции поступают через пиар-распространение ([#246](https://github.com/Adamant-im/adamant/pull/246)), устраняя реальную уязвимость отравления пула без изменения путей повторного воспроизведения консенсуса. Обновление зависимостей на нескольких этапах снижает риск известных проблем в экосистеме Node.js.

### Должны ли делегаты и операторы узлов обновляться?

Обновление рекомендуется, но не обязательно. Версия v0.10.0 не вводит обязательный консенсусный форк для сетей, которые уже синхронизированы и работают нормально; поведение, зависящее от высоты блоков, остаётся управляемым через конфигурацию. Однако cryptofoundry рекомендует делегатам и независимым операторам узлов обновляться, когда это возможно. Новые мессенджеры и API ожидают наличия функций v0.10.0, таких как `timestampMs`, запросы не подтверждённых транзакций и транспорт через WebSocket. Улучшения в установке и логировании значительно упрощают повседневную эксплуатацию, а усиление безопасности приносит пользу всей сети, даже если правила консенсуса остались прежними. Оставаться на очень старых версиях узлов в конечном итоге означает поддерживать клиентов в одиночку и упускать работу над надёжностью, которую сообщество внедряет в таких релизах.

### Технические особенности

Теперь среда выполнения требует Node.js ≥ 22.13.0; поддержка Node 18 прекращена. С точки зрения API, ключевыми нововведениями являются `timestampMs`, `returnUnconfirmed`, `includeDirectTransfers` и числовое `count`. P2P-уровень получает транспорт через WebSocket и проверки временных меток при получении транзакций через gossip. Операторы выигрывают от обновлённых скриптов установки и восстановления, вспомогательных утилит localnet, сценариев live-тестирования и возможности переопределения конфигурации. Улучшения для разработчиков включают файл AGENTS.md, расширенные рекомендации CONTRIBUTING и структурированное логирование. Удалены устаревшие HTTP-эндпоинты, а домашней страницей документации стал docs.adamant.im.

Полные структурированные заметки и полный список PR доступны в [GitHub Release v0.10.0](https://github.com/Adamant-im/adamant/releases/tag/v0.10.0). Существующим операторам следует ознакомиться с инструкциями по обновлению, обновить Node.js, перейти на v0.10.0 и перезапустить узел корректно. Новые операторы должны использовать официальные скрипты установки из тега релиза. Разработчикам следует обратиться к [CONTRIBUTING.md](https://github.com/Adamant-im/adamant/blob/master/.github/CONTRIBUTING.md) для тестирования в localnet и реальных сценариях.
