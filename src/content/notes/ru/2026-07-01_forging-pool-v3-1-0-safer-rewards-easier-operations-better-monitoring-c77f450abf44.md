ADAMANT Forging Pool v3.1.0 — рекомендуемое обновление для операторов пулов. Оно улучшает расчёт вознаграждений и надёжность выплат, усиливает логику учёта, модернизирует среду выполнения и упрощает повседневную эксплуатацию.

### Почему это обновление важно

Основная задача пула форжинга — корректно рассчитывать и безопасно выплачивать вознаграждения голосующим. Версия v3.1.0 делает акцент именно на этом. Потоки расчёта и выплаты вознаграждений были пересмотрены и укреплены, чтобы снизить риски в пограничных случаях, связанных с повторными попытками, частичными сбоями и сохранёнными значениями вознаграждений. Накопленные вознаграждения нормализуются перед учётом выплат, а прогресс вознаграждений для каждого голосующего теперь отслеживается безопаснее, чтобы сбой или повтор не привёл к случайной дубликации обновлений. Это делает обновление особенно важным для операторов, которым важна точность выплат и стабильная работа в долгосрочной перспективе.

### Что нового в v3.1.0

Ключевое изменение в инфраструктуре — переход на хранилище на основе MongoDB, которое даёт операторам пулов более надёжный уровень хранения данных о блоках, голосующих, транзакциях, истории вознаграждений и операционной информации. Для существующих пулов v3.1.0 включает инструменты миграции данных, ранее основанных на LowDB, поэтому операторы могут обновиться, не потеряв историческое состояние вознаграждений.

![Forging Pool v3.1.0: Safer Rewards, Easier Operations, Better Monitoring](/images/engineering-notes/medium/c77f450abf44/002-a7aa0fc331.webp)

Обновление также добавляет более безопасное распределение вознаграждений и учёт выплат, защиту от повторов при отслеживании прогресса вознаграждений, улучшенные логи выплат и обработки блоков, эндпоинт `/api/health` для внешнего мониторинга, поддержку опционального шифрования парфразы делегата, команды CLI `adm-pool` для шифрования, разблокировки, блокировки и проверки статуса, фильтрацию на дашборде по адресу или имени, более понятное отображение голосующих и делегатов в таблицах, обновлённую документацию и базовую среду выполнения Node.js 22.13.0+.

### Повышенная безопасность для операторов

Операторы пулов теперь могут шифровать парфразу делегата с помощью пароля оператора. Эта функция не обязательна, и существующие конфигурации с открытыми парфразами продолжают поддерживаться, но новый рабочий процесс предлагает более безопасный путь для эксплуатации в продакшене. При шифровании парфразы пул может запускаться в заблокированном состоянии. Синхронизация блоков, дашборд и публичные API остаются доступными, а выплаты и уведомления ADM приостанавливаются до тех пор, пока оператор не разблокирует пул. Это означает, что сервер может восстановиться или перезапуститься, не раскрывая сразу возможность выплат.

### Упрощённые операции и мониторинг

Новый CLI `adm-pool` предоставляет операторам простые команды для выполнения наиболее чувствительных действий во время работы:

```sh
npm run adm-pool encrypt
npm run adm-pool unlock
npm run adm-pool lock
npm run adm-pool status
```

Вместо ручного управления каждым чувствительным изменением состояния через конфигурационные файлы или логи процессов, операторы получают выделенный рабочий процесс управления. Новый эндпоинт `/api/health` предоставляет снимок состояния без секретов, который могут использовать инструменты мониторинга, такие как Zabbix, пользовательские дашборды или проверки доступности. В совокупности с хранилищем MongoDB и более понятными логами это делает пул проще для наблюдения, отладки и поддержки в долгосрочной перспективе.

### Рекомендации по обновлению

ADAMANT Forging Pool v3.1.0 рекомендуется всем операторам пулов, особенно тем, кто запускает продакшен-пулы с регулярными выплатами. Перед обновлением операторам следует создать резервную копию конфигурации и истории вознаграждений, проверить настройки MongoDB, протестировать миграцию на копии существующих данных и убедиться в правильности настроек выплат после миграции.
