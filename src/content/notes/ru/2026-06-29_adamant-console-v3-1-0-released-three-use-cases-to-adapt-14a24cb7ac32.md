ADAMANT Console v3.1.0 теперь доступен на GitHub и npm. Этот выпуск синхронизирует Console с ADAMANT Node v0.10.0 и улучшает опыт разработки при использовании CLI, интеграций JSON-RPC и локальных JavaScript-обёрток. Решение предназначено для всех, кто использует ADAMANT в скриптах, ботах, инфраструктуре бирж, внутренних инструментах, системах мониторинга или автоматизации платежей.

### Что такое ADAMANT Console?

ADAMANT Console — это инструмент командной строки и JSON-RPC для взаимодействия с блокчейном ADAMANT. Он позволяет просматривать аккаунты, блоки, транзакции, чаты, делегатов и статус узла; отправлять переводы ADM и зашифрованные сообщения; работать как локальный мост JSON-RPC для сервисов, написанных на любом языке; а также подписывать транзакции локально, чтобы парсфразы никогда не передавались узлам ADAMANT. Последний момент особенно важен: Console разработан вокруг концепции локальной подписи. Ваше приложение локально формирует действие, Console локально подписывает его, и только подписанная транзакция отправляется в сеть.

### Что нового в v3.1.0

Основная цель этого выпуска — совместимость с ADAMANT Node v0.10.0. Среди заметных изменений: поддержка обновлённого поведения ответов и запросов узла, обновление до `adamant-api` v3, поддержка новой команды `node status`, расширение вспомогательных функций для чатов и транзакций, поддержка `returnUnconfirmed` при поиске транзакций, поиск делегатов по имени, публичному ключу или адресу ADAMANT, обновлённые фильтры прямых переводов с `includeDirectTransfers`, улучшенные примеры справки CLI, расширение охвата методов JSON-RPC, сгенерированная документация API и новый сайт документации Console, а также публикация пакета в npm с подтверждением происхождения через Trusted Publishing. Поддерживаемая среда выполнения — Node.js 22.13.0 или новее.

Установка или обновление:

```bash
npm install -g adamant-console
```

Затем проверьте локальную конфигурацию:

```bash
adm client version
adm node status
```

### Вариант использования: бот для операций с криптовалютой в командных процессах

Команда, управляющая сервисами, зависящими от платежей ADM или доступности узла, может использовать ADAMANT Console как небольшой локальный мост за ботом. Бот в Telegram, Discord или Slack может вызывать команды Console или методы JSON-RPC, чтобы отвечать на вопросы о состоянии узла, статусе транзакций, балансах кошельков и неподтверждённых входящих платежах.

Примеры проверок через CLI:

```bash
adm node status
adm get address U123456789
adm get transaction 123456789 returnUnconfirmed=1
adm get transactions recipientId=U123456789,limit=10
```

Это полезно для служб поддержки, каналов мониторинга, казначейства и внутреннего реагирования на инциденты. Боту не нужно детально знать протокол ADAMANT; он вызывает Console, разбирает JSON и выводит понятные сообщения для пользователей.

### Вариант использования: лицензирование приложений или контроль доступа на основе ADM

Ещё один практический сценарий — лёгкая система лицензирования. Самостоятельно размещённое приложение, торговый инструмент, аналитическая панель или сервис автоматизации может предоставлять премиум-доступ, когда пользователь отправляет ADM на платёжный адрес. Бэкенд назначает пользователю адрес депозита, отслеживает входящие транзакции, проверяет сумму и статус платежа, автоматически активирует доступ и, при необходимости, отправляет зашифрованное сообщение в ADAMANT в качестве квитанции.

Сервис может запрашивать транзакции следующим образом:

```bash
adm get transactions recipientId=U123456789,limit=20,returnUnconfirmed=1
```

Или отправлять подтверждение:

```bash
adm send message U123456789 "Your subscription is active"
```

Для крупных приложений этот же процесс может выполняться через JSON-RPC, чтобы основной бэкенд мог быть написан на PHP, Python, Go, Ruby, Java или любом другом языке, способном выполнять HTTP-запросы. Console становится локальным мостом к ADAMANT.

### Вариант использования: быстрые депозиты и выводы ADM через JSON-RPC для бирж

Биржи и кастодиальные сервисы часто нуждаются в простом и предсказуемом интерфейсе для депозитов и выводов. ADAMANT Console может работать как локальный сервер JSON-RPC:

```bash
adm rpc server
```

По умолчанию он прослушивает настроенный порт RPC, обычно `5080`. Запускайте сервер JSON-RPC только в доверенной инфраструктуре, за брандмауэром или в приватной сети. Если сервер имеет доступ к парсфразам, рассматривайте его как инфраструктуру для подписи.

Проверка статуса узла:

```bash
curl -s -X POST http://127.0.0.1:5080 \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"nodeStatus","params":[],"id":1}'
```

Создание аккаунта депозита:

```bash
curl -s -X POST http://127.0.0.1:5080 \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"accountNew","params":[],"id":2}'
```

Надёжно храните сгенерированные учётные данные. Не сохраняйте парсфразы или приватные ключи в логах.

Мониторинг депозитов:

```bash
curl -s -X POST http://127.0.0.1:5080 \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"getTransactionsReceivedByAddress","params":["U123456789"],"id":3}'
```

Для более гибкого сканирования транзакций:

```bash
curl -s -X POST http://127.0.0.1:5080 \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"getTransactions","params":["recipientId=U123456789","limit=20","returnUnconfirmed=1"],"id":4}'
```

Ваш бэкенд биржи может сверять депозиты по адресу, идентификатору транзакции, сумме, временной метке и политике подтверждений.

Обработка выводов:

```bash
curl -s -X POST http://127.0.0.1:5080 \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"sendTokens","params":{"address":"U987654321","amount":"10ADM","passphrase":"your local passphrase"},"id":5}'
```

Для производственных систем парсфразы должны поступать из защищённого локального хранилища секретов, а не из логов, скриншотов, вывода CI или общего истории командной строки.

### Почему этот выпуск важен

ADAMANT Console намеренно лёгкий. Он не пытается заменить полноценный SDK или собственный бэкенд. Вместо этого он предоставляет разработчикам и операторам практичный инструмент для быстрых скриптов, локальной подписи, интеграции ботов, автоматизации бирж, проверки платежей, операционного мониторинга и доступа к JSON-RPC из стеков, отличных от JavaScript. С версией v3.1.0 этот инструмент теперь полностью синхронизирован с ADAMANT Node v0.10.0 и текущим стеком JavaScript API ADAMANT.
