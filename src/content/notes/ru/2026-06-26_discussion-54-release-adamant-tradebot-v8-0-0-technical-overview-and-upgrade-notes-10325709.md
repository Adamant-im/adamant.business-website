## Обзор

ADAMANT tradebot v8.0.0 теперь доступен в ветке `dev` и помечен как v8.0.0. Это крупное обновление открытого (Basic) бота для маркетмейкинга приводит кодовую базу OSS в соответствие с текущей инженерной основой Premium, сохраняя при этом границы функциональности OSS: ориентация на REST, спотовые рынки, без повторного включения модулей, доступных только в Premium. Если вы используете бота для листингового токена, практический результат — более стабильная работа, улучшенный запуск, безопасные обновления существующих баз данных, обновлённые коннекторы к биржам и опциональный приватный WebUI API для внешних клиентов.

## Архитектура

Последовательность запуска в `app.js` теперь явная и упорядоченная. Она ожидает подключения к MongoDB и сигнала `db.ready`, запускает зарегистрированные миграции базы данных, инициализирует сервисы, такие как ADM-сокет/опрос и опциональный WebUI API, подготавливает метаданные коннекторов и, наконец, запускает активные модули торговли `mm_*`. Это снижает вероятность гонок условий при холодном старте и делает обновления безопаснее.

Монолитный файл `modules/commandTxs.js` был разделён на специализированные обработчики в каталоге `modules/commands/` (account, orders, trade, info, features) вместе с общими вспомогательными компонентами. Команды по-прежнему поступают через ADAMANT Messenger и Telegram (если настроено), но путь обработки стал проще для сопровождения. Устаревшие модули приёма ADM заменены на `adamantApi.js`, `admTxChecker.js` и `admTxParser.js`, что приводит приём команд в соответствие с `adamant-api` 3.x и современными шаблонами сокетов и опроса.

Миграции базы данных выполняются однократно при запуске через `modules/dbMigrations.js`. Первая включённая миграция безопасно переименовывает устаревшее поле заказа `type` в `side`. Существующие развертывания должны создать резервную копию MongoDB перед первым запуском v8, а затем проверить открытые ордера и статистику после миграции.

## Опциональный WebUI API (Bot API v1)

v8.0 включает приватный HTTP API, расположенный в каталоге `api/`. Он использует сервер Fastify с JWT-аутентификацией, валидацией запросов через Zod и Socket.IO для транспортных хуков. Основные маршруты в `/api/v1` включают возможности bootstrap, market, account, params, commands и messages. API является опциональным и по умолчанию отключён, он предназначен для использования с самохостинговыми графическими клиентами, в то время как бот остаётся единственным компонентом, взаимодействующим с биржами. Включите его через настройку `private_webui` в конфигурации. Установите надёжный `private_webui_secret_key`, ограничьте `private_webui_allowed_ips` и не размещайте API в публичном интернете, если вы не уверены в своих действиях.

## Коннекторы к биржам

Поддерживаемые коннекторы в версии Basic включают Azbit, P2PB2B, StakeCube, Coinstore, FameEX (через новый коннектор FameEXnet) и NonKYC. Обратите внимание на изменения, нарушающие обратную совместимость: для FameEX требуется переключить конфигурацию на FameEXnet, а XeggeX удалён из OSS. Модули маркетмейкинга обновлены вместе с адаптерами трейдеров.

## Зависимости и руководство по обновлению

Заметные обновления среды выполнения включают `adamant-api` 3.x, `mongodb` 7, а также добавление `zod`, `fast-jwt` и `json-parse-bigint`. Express удалён из пути WebUI. Чтобы обновиться с v7.x, остановите бота, загрузите последний код и установите зависимости:

```bash
pm2 stop tradebot
cd adamant-tradebot
git pull
npm i
```

Убедитесь, что установлен Node.js 22.2 или новее. Слейте новые поля из `config.default.jsonc` в ваш `config.jsonc`, проверьте настройки коннектора FameEXnet и создайте резервную копию базы данных MongoDB, чтобы миграции выполнились при первом запуске. Если WebUI API включён, проверьте его настройки безопасности. Наконец, перезапустите бота с помощью вашего менеджера процессов. Именованные конфигурации продолжают работать как раньше.

## Границы функциональности OSS

v8.0 переносит в кодовую базу OSS качество инженерных решений из линейки Premium, не возвращая при этом привилегированные допущения. Базовый уровень — спотовые рынки, приоритет REST — сохранён, без обязательных WebSocket-коннекторов к биржам, стека perpetual/futures или расширенного каталога бирж.
