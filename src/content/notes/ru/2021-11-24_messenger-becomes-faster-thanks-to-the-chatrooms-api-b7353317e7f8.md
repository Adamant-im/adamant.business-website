ADAMANT обрабатывает каждое сообщение как транзакцию блокчейна, что усиливает приватность и безопасность, но создает сложности для скорости и получения сообщений. Стандартные API блокчейна требовали загрузки всех транзакций для отображения переписки. Новое API Chatrooms, разработанное специально для мессенджеров, делает загрузку сообщений в десять раз эффективнее, одновременно снижая потребление памяти и процессора.

![Messenger становится быстрее благодаря API Chatrooms](/images/engineering-notes/medium/b7353317e7f8/002-0-g1gbxwpdnkihcskz.webp)

На практике загрузка аккаунта в предыдущей версии занимала 25 секунд и потребляла 80 МБ памяти. В новой версии загрузка занимает 3 секунды и использует 28 МБ памяти — увеличение скорости в восемь раз. Прирост производительности растёт пропорционально количеству сообщений в аккаунте.

Chatrooms предоставляет два эндпоинта: `/api/chatrooms/U000000000000` и `/api/chatrooms/U000000000000/U000000000001`. Первый возвращает список чатов для конкретного аккаунта, а второй — сообщения между двумя аккаунтами. Поддерживается пагинация для минимизации передачи данных, как описано в AIP 14. Эти эндпоинты могут использоваться любыми приложениями, включая мессенджеры или ботов.

Для поддержки этого функционала ноды были обновлены для обработки новых запросов приложения. В отличие от предыдущей версии, которая загружала все сообщения сразу для оффлайн-просмотра, API Chatrooms загружает сообщения по мере необходимости и требует подключения к интернету.

Выпуск v3.0.0 вводит API Chatrooms вместе с рядом других обновлений. Он заменяет HTTP-ноду, удаляет биржу Resfinex и токен RES, а также реализует обходное решение для ошибки `includePending` в Lisk Service. Дополнительные исправления устраняют проблемы с `background-color` в светлой/тёмной темах, бесконечный цикл обновления в списке транзакций ADM и обновляют зависимости.
