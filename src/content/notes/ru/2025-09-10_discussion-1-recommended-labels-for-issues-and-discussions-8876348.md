ADAMANT использует единый набор меток для Задач и Обсуждений. Хотя некоторые метки могут быть неодинаково применимы к обоим типам, общий набор обеспечивает гибкость для сообщества. Стандартные метки GitHub остаются в нижнем регистре, в то время как пользовательские начинаются с заглавной буквы (за исключением префиксов).

Встроенные метки GitHub сохраняются без изменений: `bug`, `documentation`, `duplicate`, `enhancement`, `good first issue`, `help wanted`, `invalid`, `question` и `wontfix`.

## Специальные статусы

Три пользовательские метки статуса указывают приоритет и масштаб. `High priority` обозначает срочные задачи, требующие немедленного внимания. `Primary feature` выделяет функции, критически важные для работы продукта. `Composite task` идентифицирует крупные или сложные задачи, которые разбиваются на несколько подзадач.

## Языки, фреймворки и технологии

Метки технологий отслеживают соответствующий стек: `JavaScript` и `TypeScript` — для frontend и backend кода, `Vue` — для функций на Vue.js, `CSS` — для стилей и тем, `NodeJS` — для логики бэкенда и API, `Swift` — для iOS, `Kotlin` — для Android, `IPFS` — для децентрализованного хранения, `Electron` — для упаковки десктоп-приложений, `Capacitor` — для мобильных сборок и интеграций.

## Продукт, платформа и рабочие процессы

Платформенно-специфичные метки включают `Web`, `Mobile` и `Desktop` для проблем, специфичных для клиентов, в то время как `UX/UI` охватывает улучшения интерфейса и пользовательского опыта. `Performance` касается оптимизации скорости, хранилища или сети. `Protocol & AIPs` охватывает изменения в протоколе блокчейна ADM и взаимодействие узлов. `Integration` касается подключения внешних блокчейнов, сервисов или API. `Auth` включает парфразы, ключи, passkeys, биометрию и безопасное локальное хранение. `Nodes` отслеживает программное обеспечение узлов ADM, API, подключение и консенсус. `DB` охватывает PostgreSQL, MongoDB, миграции и запросы. `Infrastructure` включает узлы монет и ADAMANT, серверы, VPS, Ansible, Nginx, Docker и Zabbix. `APIs` касается проектирования API и конечных точек. `Automation`, `CI/CD` и `Linter` охватывают скрипты, GitHub Actions, конвейеры и проверки качества кода (ESLint/Stylelint).

## Метки для обсуждений

Специфичные для обсуждений метки помогают категоризировать беседы. `Guideline` охватывает лучшие практики, стандарты кодирования и правила участия. `Blockchain` и `Cryptocurrency` касаются механики реестра и обсуждений ADM или других токенов. `Messaging` относится к мессенджерам, протоколам связи и упоминаниям. `Decentralization` и `Privacy` отслеживают усилия по снижению централизованных зависимостей и защите анонимности пользователей. `Encryption` и `Security` охватывают сквозное шифрование, криптографические алгоритмы, управление ключами и уязвимости. `Governance` касается AIP и предложений. `Community`, `Ideas` и `Research` поддерживают обратную связь, предложения по долгосрочному развитию и исследования. `Trading` охватывает ботов, рынки, биржи и ликвидность. `Memes` — для развлекательного контента сообщества. `Marketing` и `Marketing offers` касаются продвижения, роста сообщества, партнёрств, листингов на биржах и взаимодействия с KOL.
