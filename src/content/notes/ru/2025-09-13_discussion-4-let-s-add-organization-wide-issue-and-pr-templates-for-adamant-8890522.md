Для повышения согласованности во всех репозиториях ADAMANT можно воспользоваться функцией шаблонов на уровне организации в GitHub. Создав специальный репозиторий `.github` в корне организации, мы сможем задать шаблоны по умолчанию, которые автоматически наследуются репозиториями, не имеющими собственных пользовательских шаблонов.

Этот репозиторий будет содержать несколько файлов шаблонов. Для отчётов об ошибках будет использоваться файл `bug_report.yml`, который структурирует процесс сообщения об ошибках. Для запросов функций — `feature_request.yml`, который направит участников. Файл `config.yml` может управлять видимостью шаблонов и добавлять ссылки для контактов, а файл `PULL_REQUEST_TEMPLATE.md` будет стандартизировать описания запросов на внесение изменений (pull request).

Внедрение этих шаблонов обеспечит чёткую структуру для участников, гарантируя, что такие важные детали, как шаги воспроизведения, мотивация и альтернативы, не будут упущены. Это сэкономит время сопровождающих за счёт сокращения неполных задач и улучшит общий опыт разработки во всех проектах ADAMANT.

Следующим шагом является согласование окончательной формулировки и полей для этих шаблонов. После утверждения мы сможем подготовить запрос на внесение изменений с готовыми к использованию файлами.
