### Представляем ADM-Payment

Монетизация программного обеспечения не должна зависеть от хрупкой цепочки разрозненных сервисов. Для многих продуктов — особенно в Web3, трейдинге, SaaS, автоматизации и самохостируемых системах — главная сложность заключается не просто в приёме платежей, а в управлении полным коммерческим циклом: аутентификация, оформление заказа, биллинг, подписки, выдача и проверка лицензий, администрирование, управление пользователями, продления, пробные периоды и контроль доступа.

ADAMANT Payment (*ADM-Payment*) — это универсальная самохостируемая платформа с приоритетом криптовалютных платежей, предназначенная для управления платежами, подписками и лицензиями на программное обеспечение. Платформа выпущена как проект в стадии разработки, с готовой бета-версией. Она создана как независимый продукт, не привязанный к ADAMANT Messenger или какому-либо одному приложению. Первой внутренней интеграцией станет система подписок для ADAMANT Tradebot WebUI, однако платформа разрабатывается для более широкого спектра решений: ботов, SaaS-платформ, десктопных приложений, Web3-сервисов, приватных инструментов, коммерческих API и самохостируемых продуктов.

### Зачем это было создано

Большинство современных систем монетизации изначально ориентированы на традиционные фиатные платежи. Это подходит многим бизнесам, но становится ограничением, если продукт крипто-нативный, глобальный, самохостируемый, ориентирован на приватность или ориентирован на пользователей, не желающих использовать традиционные банковские каналы. Типичная конфигурация может включать отдельные сервисы для аутентификации, платежей, подписок и лицензий, плюс кастомные скрипты для контроля доступа, самописную админ-панель, вебхуки, колбэки, базы данных и ручные процессы поддержки.

ADM-Payment объединяет все эти компоненты в единую согласованную платформу. Владелец продукта может определять тарифные планы, принимать криптовалютные платежи, выдавать лицензии, управлять пользователями и позволять внешнему ПО проверять доступ через API — без необходимости каждый раз пересоздавать весь слой монетизации.

### Основные модули

Платформа объединяет в одном самохостируемом решении аутентификацию, биллинг, криптовалютные платежи, пробные и платные лицензии, подписки, промо- и ручные лицензии, веб-портал для пользователей, административную панель, API проверки лицензий, брендирование, поддержку локализации и функции операционной безопасности.

Платежи изначально ориентированы на криптовалюты, а не добавлены как опция в последний момент. На текущий момент поддерживаются платежи в Bitcoin через самохостируемый BTCPay Server, нативные платежи ADAMANT с уникальными адресами депозитов и отслеживанием в блокчейне, а также тестовый провайдер для отладки процессов. Архитектура позволяет в будущем добавлять новые провайдеры (Ethereum, ERC20, стейблкоины и др.) и блокчейны без переписывания ядра биллинга.

Для пользователей ADM-Payment предоставляет понятный веб-интерфейс для входа, просмотра каталога, оформления заказа и управления лицензиями. Для владельцев продуктов — административную панель для управления аккаунтами, лицензиями, счетами, кошельками, тарифами и правами доступа. Для внешних приложений — API проверки лицензий, позволяющее ботам, бэкендам SaaS, релеям, десктопным приложениям и другим продуктам программно проверять, активен ли доступ у пользователя.

### Целевые сценарии использования

ADM-Payment — это слой монетизации, а не просто страница оплаты. Особенно полезна она для трейдинг-ботов и инструментов автоматизации, которым требуется лицензионный доступ, подписки, ограничения по биржам или рынкам и приватные развертывания. Разработчик трейдинг-бота может создать тарифы Basic, Pro или Enterprise; пользователи платят в криптовалюте, получают лицензии, а бот проверяет доступ через API. Лицензии могут быть ограничены по параметрам, специфичным для продукта — например, по бирже или торговой паре, что обеспечивает точный контроль доступа, выходящий за рамки простой модели «оплачено / не оплачено».

Для SaaS-продуктов с крипто-нативной аудиторией ADM-Payment позволяет принимать криптовалюту, управлять подписками и контролировать доступ без полной зависимости от традиционных платежных процессингов. Десктопные приложения и приватные инструменты могут использовать платформу как бэкенд лицензирования, вызывая API проверки для подтверждения активности лицензии. Web3-сервисы получают выгоду от аутентификации через крипто-кошельки, включая вход через ADM и Ethereum-кошельки, наряду с классическим email-логином. Коммерческие самохостируемые продукты могут развернуть и адаптировать платформу под свои правила вместо зависимости от закрытых SaaS-решений для лицензирования.

Платформа поддерживает логику пробных периодов с автоматической регистрацией и правилами, например, один пробный период на заданную область. Также доступны типы лицензий: платные, пробные, промо- и ручные, что даёт операторам гибкость без необходимости прямого редактирования базы данных. Брендирование настраивается через переменные окружения, а модель данных использует универсальные идентификаторы продуктов вместо жёстко заданных предположений о ADAMANT, что делает платформу пригодной для white-label-развёртываний в разных продуктах.

### Техническая архитектура

ADM-Payment построена как современный монорепозиторий с использованием `pnpm` и Turborepo. Версия v1.0.0 включает бэкенд на Fastify 5 с ORM Prisma, базу данных PostgreSQL, фронтенд на React 18 с Vite, отдельные веб-приложения для пользователей и администраторов, общие пакеты с общей логикой, JWT-сессии с refresh-куками, аутентификацию через коды сообщений ADM, аутентификацию Ethereum через SIWE, вход по email и паролю, капчу Turnstile, интеграцию с BTCPay Server для приёма Bitcoin, нативный провайдер платежей ADAMANT с уникальными адресами депозитов и on-chain наблюдателем, идемпотентные вебхуки, API-ключ администратора, опциональную двухфакторную аутентификацию, блокировку по IP и отпечаткам, аудит-логирование, поддержку локализации i18n и CI на GitHub Actions для установки, генерации Prisma, сборки, линтинга и проверки типов.

Архитектура разделяет пользовательскую и административную зоны. Внешние продукты взаимодействуют с платформой через API-эндпоинты для проверки лицензий или статуса подписки. Такой подход «API-first» означает, что ADM-Payment — это не просто страница оформления заказа, а бэкенд-сервис, на который могут полагаться другие приложения.

Контроль безопасности реализован на уровне архитектуры: разделение пользовательской и административной зон, защищённые потоки аутентификации, контроль доступа, безопасные API проверки лицензий, API-ключ администратора, опциональная двухфакторная аутентификация ADM и ETH, поддержка капчи, блокировка по IP и отпечаткам, аудит-логирование. Аутентификация через кошельки позволяет пользователям использовать крипто-идентичность вместо обязательного email-аккаунта, при этом вход по email и паролю остаётся доступным для традиционных пользователей.

### Текущее состояние

ADM-Payment находится в стадии разработки, с готовой бета-версией. Основа уже пригодна для использования, но продукт продолжает совершенствоваться: дорабатываются пользовательские сценарии, расширяется документация и собирается обратная связь от реальных интеграций. Текущее направление развития — фундамент платформы v1.0.0, с первой производственной интеграцией для подписок ADAMANT Tradebot WebUI. В дорожной карте — добавление новых провайдеров платежей, поддержка дополнительных блокчейнов, документация OpenAPI, автоматизация продления подписок и интеграции с продуктами третьих сторон.
