ADAMANT Forging Pool v3.1.0 — рекомендуемое обновление для операторов пулов. Релиз исправляет и улучшает расчёты вознаграждений и обработку запланированных выплат, а также вносит несколько структурных изменений, которые стоит изучить перед обновлением рабочих пулов.

## Почему нужно обновиться

Наиболее важные изменения касаются безопасности выплат, безопасности оператора и долгосрочной поддерживаемости. Начисленные вознаграждения теперь нормализуются перед учётом выплат, а прогресс вознаграждений по каждому голосующему сохраняется для сгенерированных блоков. Пути повторных попыток и аварийного завершения стали безопаснее с точки зрения дублирования обновлений вознаграждений. Поведение выплат, разбора блоков и хранилища теперь покрыто более целенаправленными тестами, а логи и уведомления стали понятнее при сбоях в работе.

## Хранение данных и миграция

Теперь хранилище пула основано на MongoDB. В релизе включены инструменты миграции для старых данных пулов на основе LowDB, расположенные в `scripts/migrate-lowdb-mongodb/`, а также тесты миграции, настройка индексов MongoDB, обновлённая конфигурация по умолчанию и руководство по миграции в README. Операторам следует создать резервную копию существующих данных пула, протестировать миграцию на копии и проверить итоги начисленных и полученных вознаграждений перед переключением рабочих выплат на обновлённый пул.

## Безопасность оператора

Версия v3.1.0 добавляет поддержку опционального шифрования парфразы. Конфигурации с незашифрованной парфразой по-прежнему поддерживаются, но теперь операторы могут зашифровать парфразу делегата и разблокировать работающий пул только тогда, когда нужно включить выплаты:

```sh
npm run adm-pool encrypt
npm run adm-pool unlock
npm run adm-pool lock
npm run adm-pool status
```

Интерфейс управления использует локальный Unix-сокет, доступный только владельцу. При использовании зашифрованной парфразы пул может запускаться в заблокированном состоянии: синхронизация блоков, панель управления и публичное API остаются доступными, а выплаты и уведомления ADM приостанавливаются до разблокировки.

## Мониторинг и панель управления

Релиз добавляет эндпоинт `/api/health` для внешнего мониторинга без секретов, а также статус заблокированных выплат на панели. Теперь поддерживается фильтрация голосующих и транзакций по адресу или имени, а имена делегатов и голосующих отображаются под адресами, если доступны. Также включены исправления сортировки числовых значений, улучшенные ссылки на обозреватель и улучшения компоновки панели управления.

## Примечания по запуску и обновлению

Теперь базовая среда выполнения — Node.js 22.13.0+ и npm 10+. Файлы README, CONTRIBUTING и руководства в репозитории обновлены для настройки, миграции, мониторинга, безопасности секретов и процесса вклада в разработку.

Перед обновлением создайте резервную копию конфигурации и истории вознаграждений, проверьте настройки MongoDB в `config.default.jsonc` и протестируйте миграцию с LowDB на MongoDB на копии данных. После миграции убедитесь в корректности сумм начисленных и полученных вознаграждений, проверьте настройки кошельков для выплат и обслуживания, соберите и протестируйте панель управления, а также проверьте `/api/health`. Включайте выплаты только после проверки логов запуска и статуса пула.
