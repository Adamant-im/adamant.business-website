ADAMANT Forging Pool v3.1.0 — рекомендуемое обновление для операторов пулов.

Этот выпуск исправляет и усиливает пути расчёта вознаграждений и учёта выплат, переносит хранилище пула в коллекции с поддержкой MongoDB и добавляет инструменты миграции данных старых пулов из LowDB в MongoDB. Введено отслеживание прогресса выплат вознаграждений по каждому участнику с защитой от повторных выплат после сбоев или частичного выполнения, нормализация сохранённых отложенных вознаграждений перед учётом выплат, а также добавлена опциональная поддержка шифрования парфразы пула. Новый операторский CLI `adm-pool` предоставляет команды `encrypt`, `unlock`, `lock` и `status`, а также локальный сокет управления только для владельца для команд блокировки/разблокировки во время работы. Эндпоинт `/api/health` позволяет осуществлять внешний мониторинг без использования секретов. На панели управления теперь доступны фильтрация участников и транзакций, отображение имён делегатов и участников, исправления численной сортировки и улучшенные ссылки. Документация обновлена для настройки, конфигурации, миграции, мониторинга и рабочих процессов операторов. Базовая версия среды выполнения обновлена до Node.js 22.13.0+ и npm 10+.

Это рекомендуемое обновление, поскольку включает исправления и меры защиты для расчёта вознаграждений и запланированных выплат. Перед обновлением операторам следует создать резервную копию текущей конфигурации пула и истории вознаграждений, проверить конфигурацию MongoDB в файле `config.default.jsonc`, ознакомиться с руководством по миграции в `README.md`, протестировать процесс миграции на копии существующих данных и проверить настройки выплат, пожертвований и кошелька технического обслуживания после миграции. Шифрование парфразы является опциональным; конфигурации с открытым текстом продолжают поддерживаться. Операторы могут зашифровать парфразу делегата с помощью:

```sh
npm run adm-pool encrypt
```

Затем разблокировать работающий пул командой:

```sh
npm run adm-pool unlock
```
