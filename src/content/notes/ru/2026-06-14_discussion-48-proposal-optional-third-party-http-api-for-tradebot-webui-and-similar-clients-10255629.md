Это предложение вводит необязательное Bot API v1, позволяющее внешним клиентам — например, графическому веб-интерфейсу (WebUI) — отслеживать и управлять торговым ботом без зависимости от ADAMANT Messenger или Telegram. В данной архитектуре веб-интерфейс обменивается данными исключительно с API бота, обеспечивая, что бот остаётся единым источником истины для рыночных данных (тикеры, стаканы, сделки, OHLC), состояния счёта, параметров торговли и выполнения команд.

API призван обеспечить необязательный входящий HTTP и WebSocket интерфейс на стороне бота, используя порт `private_webui` в конфигурации. Аутентификация будет осуществляться через JWT, проверяемый ботом с использованием `private_webui_secret_key`, при этом учётные записи пользователей будут храниться во внешнем клиенте. API предоставит структурированные JSON-эндпоинты для рыночных данных, данных аккаунта и параметров, а также обёртки для команд. Эндпоинт `GET /bot` будет использоваться для начальной загрузки, возвращая возможности, такие как установленные модули `mm_*.js` и флаги функций бирж. Для динамического изменения параметров торговли будет доступно событие WebSocket `params:updated`. Транспортные режимы изначально будут поддерживать `directHttp` для самостоятельно размещённых клиентов, а режим `relayWs` для хостинга по подписке запланирован на более поздний срок.

В данном обсуждении не рассматриваются пользовательские интерфейсы оплаты, выставления счётов или лицензирования, а также детали конкретной реализации WebUI. Сообщество приглашается высказать мнение о том, следует ли размещать это API в открытом репозитории `adamant-tradebot` в качестве необязательного модуля, какие эндпоинты необходимы для минимального полезного клиента помимо команд мессенджера, а также предпочтительный способ получения рыночных и балансовых данных — опрос (polling) или push — в версии v1. В репозитории `adamant-tradebot` будет создана отслеживающая задача (tracking issue) с ссылкой на это обсуждение. Премиум-реализация уже разрабатывается в ветке `adamant-tradebot-me` под названием `refactor/new-webui-api`.
