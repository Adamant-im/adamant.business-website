Cette version introduit le système de stockage clé-valeur (KVS), également appelé états (states), qui est nécessaire pour implémenter des fonctionnalités supplémentaires dans les applications clientes. Les frais pour les opérations KVS sont les mêmes que pour les messages. De nouvelles extrémités d'API (endpoints) ont été ajoutées pour prendre en charge ces fonctionnalités, notamment l'extrémité `/states/` avec les actions `store` et `get`.

Les nouvelles extrémités ne prennent pas en charge le chiffrement côté nœud, donc toutes les signatures doivent être effectuées côté client. Un exemple d'utilisation est disponible dans le package adamant-console.

### Changements incompatibles

Cette version introduit des changements incompatibles dans l'API, ce qui explique l'incrémentation de la version mineure. L'ajout du système KVS et de ses extrémités associées modifie l'interface API existante, obligeant les applications clientes à s'adapter aux nouvelles extrémités et aux exigences de signature côté client.
