Cette version introduit plusieurs améliorations d'API et correctifs pour le logiciel de nœud ADAMANT. Le point d'entrée `/api/delegates/get` accepte désormais un paramètre `address` en plus des paramètres précédemment pris en charge `username` et `publicKey`. L'utilisation est simple :

```bash
/api/delegates/get?address=U1234567890
```

Le support WebSocket a été amélioré pour permettre l'abonnement à plusieurs types de transactions au lieu d'un seul. Pour les transactions de messages, un nouvel événement `assetChatTypes` permet aux clients de s'abonner à des types de messages spécifiques.

Le point d'entrée `/api/node/status` renvoie désormais des propriétés supplémentaires en fusionnant les données provenant de `/api/loader/status` et `/api/loader/status/sync`, réduisant ainsi le nombre de requêtes nécessaires pour obtenir les informations complètes sur l'état de la blockchain :

```json5
{
  // ...
  "loader": {
    "loaded": true,
    "now": 45345521,
    "syncing": false,
    "consensus": 100,
    "blocks": 0,
    "blocksCount": 0
  }
}
```

Plusieurs bogues ont été corrigés. Un problème de connexion avec les pairs dont les noms de version incluent des métadonnées de pré-version ou de build (comme `+alpha` ou `-dev.0`) a été résolu, assurant une conformité complète avec SemVer 2.0. La suppression des pairs après des erreurs de connexion ponctuelles a été corrigée, et la validation du fichier de configuration a été réparée.

Le point d'entrée `/delegates/get` a été optimisé pour les recherches de transactions uniques. Les dépendances ont été mises à jour, de légères améliorations de journalisation ont été apportées, et un nouveau guide pour les contributeurs concernant les tests a été ajouté.
