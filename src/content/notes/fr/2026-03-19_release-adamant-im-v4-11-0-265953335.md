Cette version introduit une version universelle pour macOS et migre le projet vers une nouvelle configuration ESLint. Un fichier de notes de version a été ajouté au dépôt. Le système de contrôle d'intégrité a été amélioré : un statut « Mise à jour en cours… » est désormais affiché avant les tentatives de requête HTTP, les vérifications initiales et ultérieures suivent un algorithme de requête optimisé, et le système vérifie `alt_ip` lorsque le domaine web n'est pas accessible. La récupération en arrière-plan après la sortie du mode veille a été corrigée, et les erreurs de rapport d'état de nœud ont été résolues. L'écran des portefeuilles a été mis à jour, ainsi que le script pour adamant-wallets, et un écran de test de développement a été créé pour cet outil.

Les messages peuvent désormais être regroupés en bulles. Plusieurs mises à jour de style ont été appliquées dans toute l'application, notamment les styles de saisie de messages, la disposition du formulaire d'exportation des clés et des améliorations générales de la cohérence visuelle. Le chat de bienvenue dans ADAMANT inclut désormais un champ de saisie désactivé. Des vérifications Playwright réservées aux développeurs ont été ajoutées dans le cadre d'un effort plus large visant à unifier les styles et à moderniser l'interface utilisateur et l'expérience utilisateur.

De nombreuses corrections de bogues sont incluses. Les dates dans les discussions se rafraîchissent désormais correctement. Les indicateurs de synchronisation erronés n'apparaissent plus lors de l'ouverture de l'écran des nœuds. Les montants incorrects des transactions pour BTC, DOGE et DASH lors de l'envoi à son propre adresse ont été corrigés. Le balisage qui se rompait lorsqu'on répondait à un message contenant un lien long a été corrigé. Une erreur de référence nulle liée à `maintainScrollPosition` a été résolue. Les discussions statiques ne sont plus en lecture seule après une nouvelle connexion. Les frais de transaction pour les jetons ERC20 sont désormais cohérents entre la PWA et iOS. La navigation dans les transactions et les régressions d'interface localisée sur les écrans PWA ont été corrigées. Les symboles de portefeuille persistants hérités sont désormais normalisés pour éviter un blocage sur le chemin `/options`. Le nom de l'APK publié a été corrigé, et les problèmes de code source détectés par ESLint 9 ont été résolus. L'activité de commit erronée dans le dépôt a été corrigée.

Les dépendances et outils généraux ont été mis à jour, les avertissements de compilation ayant été supprimés. Node.js a été mis à jour de la version 20 à la version 22. Un Release Drafter organisationnel et un workflow réutilisable ont été mis en œuvre. Un workflow GitHub Actions et un hook git Husky postinstall pour ESLint ont été ajoutés. La documentation a été enrichie avec des directives de fonctionnement pour les agents IA destinées à la PWA, et le README ainsi que les métadonnées de version ont été actualisés.

### Changements cassants

Le support de Klayr (KLY) a été entièrement supprimé du portefeuille. Les utilisateurs détenant des actifs KLY doivent être informés que cette cryptomonnaie n'est plus prise en charge à partir de la version 4.11.0.
