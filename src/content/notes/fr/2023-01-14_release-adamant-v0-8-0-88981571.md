Cette version d'ADAMANT introduit une option `cors` dans `config.json`. Les utilisateurs peuvent désormais utiliser soit `inId` soit `isIn` dans les endpoints `/api/transactions` et `/api/chats/get`. Par exemple, `/api/chats/get?InId=U6386412615727665758` et `/api/chats/get?isIn=U6386412615727665758` sont équivalents.

Les identifiants d'utilisateur (User IDs) dans les endpoints `/api/chats/get` et `/api/chatrooms` sont désormais insensibles à la casse. Par exemple, `/api/chatrooms/U6386412615727665758` et `/api/chatrooms/u6386412615727665758` sont identiques.

Les transactions et `POST /api/accounts/delegates` acceptent désormais une transaction soit comme un objet brut, soit imbriquée dans la propriété `transaction`.

```bash
curl -X POST https://endless.adamant.im/api/transactions/process
   -H 'Content-Type: application/json'
   -d '{"type": 0, "amount": 100000000, ...}'  # or {"transaction": { "type": 0, "amount": 100000000, ... }}
```

L'endpoint `/api/states/get` inclut désormais une propriété `confirmations`. Une optimisation de requête `generatorPublicKey` a été appliquée. Une erreur « permission denied for schema public » a été corrigée, ainsi que d'autres modifications mineures.

### Changements cassants

L'endpoint `/api/blocks` ne renvoie plus la propriété `count`.
