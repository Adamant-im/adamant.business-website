ADAMANT Forging Pool v3.1.0 est une mise à jour recommandée pour les opérateurs de pool.

Cette version corrige et renforce les chemins de calcul des récompenses et de comptabilisation des paiements, déplace le stockage du pool vers des collections basées sur MongoDB, et ajoute des outils de migration de LowDB vers MongoDB pour les anciennes données de pool. Elle introduit un suivi de progression des récompenses par votant résistant aux tentatives multiples, afin d'éviter les récompenses en double après un incident ou une exécution partielle, normalise les récompenses en attente avant la comptabilisation des paiements, et ajoute un support optionnel pour les phrases secrètes chiffrées du pool. Le nouveau CLI opérateur `adm-pool` fournit les commandes `encrypt`, `unlock`, `lock` et `status`, ainsi qu’un socket de contrôle local accessible uniquement par le propriétaire pour les commandes de verrouillage/déverrouillage à l’exécution. Un point de terminaison `/api/health` permet une surveillance externe sans secret. Le tableau de bord inclut désormais des filtres pour les votants et les transactions, les noms des délégués et votants, des corrections de tri numérique, et des liens améliorés. La documentation a été mise à jour pour la configuration, la migration, la surveillance et les flux de travail des opérateurs. La base d'exécution est désormais Node.js 22.13.0+ et npm 10+.

Cette mise à jour est recommandée car elle inclut des correctifs et des protections pour le calcul des récompenses et les paiements planifiés. Avant la mise à niveau, les opérateurs doivent sauvegarder la configuration actuelle du pool et l'historique des récompenses, vérifier la configuration MongoDB dans `config.default.jsonc`, consulter le guide de migration dans `README.md`, tester le processus de migration sur une copie des données existantes, puis vérifier les paramètres de paiement, de dons et du portefeuille de maintenance après la migration. Les phrases secrètes chiffrées sont facultatives ; les configurations existantes avec phrases secrètes en clair restent prises en charge. Les opérateurs peuvent chiffrer la phrase secrète du délégué avec :

```sh
npm run adm-pool encrypt
```

Puis déverrouiller un pool en cours d'exécution avec :

```sh
npm run adm-pool unlock
```
