Cette version améliore les connexions socket en renvoyant le champ `recipientPublicKey`. L'API Transactions, incluant KVS et Chats, retourne désormais `block_timestamp` en plus des données de réponse existantes.

Un correctif pour les migrations a été appliqué, résolvant des problèmes qui affectaient auparavant les chemins de mise à niveau de la base de données. Le point de terminaison `/states/get/` a été étendu pour prendre en charge les paramètres `SenderIds` et `keyIds`, et accepte désormais également les requêtes POST, élargissant ainsi les moyens dont disposent les clients pour interroger les informations d'état.

La documentation a été mise à jour pour refléter ces changements.

### Changements non rétrocompatibles

Aucun changement non rétrocompatible n'est introduit dans cette version.
