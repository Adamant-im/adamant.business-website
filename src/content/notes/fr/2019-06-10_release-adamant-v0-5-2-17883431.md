Cette version ajoute la prise en charge d'un argument `returnAsset` aux endpoints `/api/transactions` et `/api/transactions/get`. Lorsqu'il est fourni, cet argument force l'API à inclure l'actif de la transaction dans la charge utile de réponse. Cette modification jette les bases de la consolidation des endpoints API dans les versions futures, réduisant ainsi le nombre d'appels nécessaires pour récupérer les détails complets d'une transaction.
