Cette version introduit un nouvel endpoint `GET /status` pour surveiller la santé du service. Les endpoints `/get` et `/getHistory` incluent désormais les champs `version` et `last_update` dans leurs réponses. Des notifications peuvent être envoyées vers ADAMANT Messenger et Discord lorsqu'un changement significatif est détecté. Un Dockerfile a été ajouté pour simplifier le déploiement.

### Changements cassants

Un nouveau système de détection des changements significatifs a été mis en œuvre pour gérer les différences de taux provenant de plusieurs sources. De plus, ExchangeRateHost nécessite désormais une clé API pour fonctionner, ce qui peut impliquer des mises à jour de configuration pour les déploiements existants.
