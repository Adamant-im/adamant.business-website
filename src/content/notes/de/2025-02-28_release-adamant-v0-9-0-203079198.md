Diese Version führt mehrere API-Verbesserungen und Fehlerbehebungen für die ADAMANT-Node-Software ein. Der `/api/delegates/get`-Endpunkt akzeptiert nun zusätzlich zu den bisher unterstützten Parametern `username` und `publicKey` auch einen `address`-Parameter. Die Nutzung ist unkompliziert:

```bash
/api/delegates/get?address=U1234567890
```

Die WebSocket-Unterstützung wurde erweitert, um das Abonnieren mehrerer Transaktionstypen statt nur eines zu ermöglichen. Bei Nachrichtentransaktionen ermöglicht das neue `assetChatTypes`-Event Clients, sich auf bestimmte Nachrichtentypen zu abonnieren.

Der `/api/node/status`-Endpunkt gibt nun zusätzliche Eigenschaften zurück, indem er Daten aus `/api/loader/status` und `/api/loader/status/sync` zusammenführt. Dadurch wird die Anzahl der Anfragen reduziert, die benötigt werden, um den vollständigen Blockchain-Status abzurufen:

```json5
{
  // ...
  "loader": {
    "loaded": true,
    "now": 45345521,
    "syncing": false,
    "consensus": 100,
    "blocks": 0,
    "blocksCount": 0
  }
}
```

Mehrere Fehler wurden behoben. Ein Verbindungsproblem mit Peers, deren Versionsnamen Pre-Release- oder Build-Metadaten enthalten (wie `+alpha` oder `-dev.0`), wurde behoben, wodurch nun vollständige SemVer 2.0-Konformität erreicht ist. Die vorübergehende Entfernung von Peers nach einmaligen Verbindungsfehlern wurde korrigiert, und die Validierung der Konfigurationsdatei wurde repariert.

Der `/delegates/get`-Endpunkt wurde für Einzeltransaktionssuchen optimiert. Abhängigkeiten wurden aktualisiert, kleinere Verbesserungen an der Protokollierung vorgenommen, und eine neue Anleitung für Testbeiträge wurde hinzugefügt.
