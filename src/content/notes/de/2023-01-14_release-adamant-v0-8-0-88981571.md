Diese Version von ADAMANT führt eine `cors`-Option in `config.json` ein. Nutzer können nun entweder `inId` oder `isIn` in den Endpunkten `/api/transactions` und `/api/chats/get` verwenden. Zum Beispiel sind `/api/chats/get?InId=U6386412615727665758` und `/api/chats/get?isIn=U6386412615727665758` äquivalent.

Die Benutzer-IDs in den Endpunkten `/api/chats/get` und `/api/chatrooms` sind nun nicht mehr auf Groß-/Kleinschreibung angewiesen. Zum Beispiel sind `/api/chatrooms/U6386412615727665758` und `/api/chatrooms/u6386412615727665758` identisch.

Transaktionen und `POST /api/accounts/delegates` akzeptieren nun eine Transaktion entweder als einfaches Objekt oder innerhalb der `transaction`-Eigenschaft.

```bash
curl -X POST https://endless.adamant.im/api/transactions/process
   -H 'Content-Type: application/json'
   -d '{"type": 0, "amount": 100000000, ...}'  # or {"transaction": { "type": 0, "amount": 100000000, ... }}
```

Der Endpunkt `/api/states/get` enthält nun eine `confirmations`-Eigenschaft. Eine Abfrageoptimierung für `generatorPublicKey` wurde implementiert. Ein „Zugriff verweigert für Schema public“-Fehler wurde behoben, zusammen mit weiteren kleineren Änderungen.

### Breaking changes

Der Endpunkt `/api/blocks` gibt die Eigenschaft `count` nicht mehr zurück.
