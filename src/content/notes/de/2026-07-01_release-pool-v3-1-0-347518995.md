ADAMANT Forging Pool v3.1.0 ist ein empfohlenes Update für Pool-Operatoren.

Diese Version behebt und stärkt die Pfade zur Belohnungsberechnung und Auszahlungsbuchhaltung, verlegt den Pool-Speicher auf MongoDB-gestützte Collections und fügt Werkzeuge für die Migration von LowDB zu MongoDB für ältere Pooldaten hinzu. Es wird eine wiederverbindungsichere, pro Wähler basierende Fortschrittsverfolgung der Belohnungen eingeführt, um doppelte Belohnungen nach Abstürzen oder teilweisen Durchläufen zu verhindern, außerdem wird die Normalisierung gespeicherter ausstehender Belohnungen vor der Buchhaltung der Auszahlungen durchgeführt. Zudem wird optionaler verschlüsselter Passphrasensupport für den Pool hinzugefügt. Die neue `adm-pool` Operator-CLI bietet die Befehle `encrypt`, `unlock`, `lock` und `status` sowie einen lokalen, nur für den Besitzer zugänglichen Steckplatz für Laufzeitbefehle zum Sperren/Entsperren. Ein `/api/health`-Endpunkt ermöglicht die externe Überwachung ohne Geheimnisse. Das Dashboard enthält nun Filter für Wähler und Transaktionen, Namen von Delegierten/Wählern, Korrekturen bei numerischer Sortierung und verbesserte Links. Die Dokumentation wurde aktualisiert für Einrichtung, Konfiguration, Migration, Überwachung und Operator-Workflows. Die Laufzeitbasis wurde auf Node.js 22.13.0+ und npm 10+ aktualisiert.

Dieses Update wird empfohlen, da es Korrekturen und Sicherheitsmaßnahmen für die Belohnungsberechnung und geplanten Auszahlungen enthält. Vor der Aktualisierung sollten Betreiber die aktuelle Poolkonfiguration und den Belohnungsverlauf sichern, die MongoDB-Konfiguration in `config.default.jsonc` prüfen, die Migrationsanleitung in `README.md` lesen, den Migrationsablauf auf einer Kopie der bestehenden Daten testen und nach der Migration die Auszahlungseinstellungen, Spendenoptionen und Wartungswallet-Einstellungen überprüfen. Verschlüsselte Passphrasen sind optional; bestehende Konfigurationen mit unverschlüsselten Passphrasen werden weiterhin unterstützt. Betreiber können die Delegat-Passphrase verschlüsseln mit:

```sh
npm run adm-pool encrypt
```

Anschließend kann ein laufender Pool entsperrt werden mit:

```sh
npm run adm-pool unlock
```
