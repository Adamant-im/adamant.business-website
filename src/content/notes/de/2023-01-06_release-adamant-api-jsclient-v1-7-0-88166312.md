Diese Version führt die Methode `api.setStartupCallback()` ein, die Entwicklern ermöglicht, einen Rückruf zu registrieren, der während des Client-Startvorgangs ausgeführt wird. Außerdem akzeptiert der API-Konstruktor nun einen Rückruf als dritten Parameter und bietet so eine bequeme Möglichkeit, Startlogik direkt zum Zeitpunkt der Instanziierung bereitzustellen.
