Um Issues innerhalb der ADAMANT-Organisation klar und einheitlich zu halten, verwenden Sie kurze Präfixe im Titel, wenn Sie ein neues Issue erstellen. Dies hilft allen, die Art der Aufgabe auf einen Blick schnell zu erfassen.

## Häufige Präfixe

Verwenden Sie **[Bug]** für Fehler, Abstürze oder unerwartetes Verhalten. Verwenden Sie **[Feat]** für neue Funktionen und **[Enhancement]** für Verbesserungen bestehender Funktionen. Verwenden Sie **[Refactor]** für Code-Refactorings, die die Funktionalität nicht ändern. Verwenden Sie **[Docs]** für Dokumentation, Anleitungen oder README-Aktualisierungen. Verwenden Sie **[Test]** zum Hinzufügen oder Verbessern von Tests. Verwenden Sie **[Chore]** für Routineaufgaben wie Abhängigkeitsaktualisierungen oder CI/CD-Anpassungen.

## Projektbezogene Präfixe

Verwenden Sie **[Task]** für eine allgemeine Aufgabe, die nicht direkt mit Programmierung zusammenhängt. Verwenden Sie **[Composite]** für eine Aufgabe, die mehrere Unteraufgaben umfasst. Verwenden Sie **[UX/UI]** für Änderungen an Benutzeroberfläche oder Benutzererfahrung.

## Ideen und Vorschläge

Verwenden Sie **[Proposal]** für formelle Vorschläge, **[Idea]** für Ideen, die in Betracht gezogen oder weiter diskutiert werden sollen, und **[Discussion]** für Themen, die weiteren Input erfordern könnten. Die beste Plattform für diese drei Kategorien ist jedoch das [ADAMANT Forum](https://github.com/orgs/Adamant-im/discussions) und nicht Issues.

## Beispiele

```
[Bug] Crash when syncing wallet on iOS
[Feature] Add support for biometric auth
[Docs] Update installation guide for Ubuntu 24
[Refactor] Split TradeBot config module
[Composite] Add decentralized file transfer across ADM messenger apps
[UI/UX] Improve Home screen design
[Task] Release iOS app v3.10.1 to the App Store
```

Verwenden Sie maximal ein oder zwei Präfixe, um den Titel nicht zu überladen.
