トレーディングソフトウェアは最悪の瞬間に評価されます。取引所 API が劣化し、WebSocket が無音で切断し、部分約定が再起動の途中で発生する。私たちはその瞬間のためのシステムを構築します — セルフホスト、可観測、初回コミットからチームのコントロール下に。

## 典型的な構築内容

- **取引所接続** — 利用する CEX と DEX 向けの REST および WebSocket コネクタ。再接続、クロックドリフト処理、取引所ごとのレート制限バジェット
- **注文エンジン** — 再起動を耐える注文ライフサイクル追跡。すべての注文は取引所と照合され、決して仮定されない
- **リスクレール** — 戦略コードと取引所の間に位置するハードポジション上限、想定元本制限、キルスイッチ、サニティチェック
- **マーケットデータパイプライン** — 複数取引所にわたる正規化された約定、板情報、ローソク足。アナリストがクエリできる場所に保存
- **運用ビュー** — ダッシュボードとアラート。人間が常にシステムの動作を把握し、一操作で停止できる

## マーケットメイキングに特化

10年間の自社マーケットメイキングソフトウェア運用が [adamant-tradebot](https://github.com/Adamant-im/adamant-tradebot) に結実しています — 無料オープンソース版と、[marketmaking.app](https://marketmaking.app) でのホスト型プレミアム体験を備えた専用マーケットメイキング・トレーディングソフトウェア。自社ハードウェア上で必要なチーム向け — カスタムペア、カスタム取引所、カスタムリスクルール — 同じエンジンのセルフホストデプロイを調整します。

## トレジャリーと実行ツール

すべてが戦略ではありません。ファンドやプロダクト会社は、地味なレイヤーで私たちに依頼します。取引所間のリバランス、大口ポジションの TWAP 型実行、ペイアウトバッチ処理、サトシ単位で照合するレポート。自動化が最も早く元を取る領域です。
