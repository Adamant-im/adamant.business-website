ボットとは、お金にアクセスできる小さなプログラムです。この枠組みがすべての意思決定を導きます。最小限の鍵権限、堅実な信頼性、完全なソースコードをお客様の手に。私たちはソフトウェアを納品します — 鍵、戦略、責任はお客様が保持します。

## 自動化する内容

- **通知・アラートボット** — 残高、ノード健全性、取引所注文状態、オンチェーン送金を監視し、損失になる前に人間に通知する Telegram および ADAMANT ボット
- **実行ヘルパー** — ボットがトランザクションや注文を準備し、人間が承認する半自動ワークフロー。トレジャリー運用や OTC 決済に有用
- **運用ダッシュボード** — チームがすでに運用しているウォレット、ノード、ボットを単一ビューで把握。8つのブラウザタブとスプレッドシートは不要
- **チャット内サービス** — 暗号化 ADAMANT チャット内で稼働するボット：取引所フロー、ペイアウト、サポートキュー、アクセス制御
- **AI 支援ツール** — LLM ベースの要約・トリアージヘルパー。常にシニアエンジニアのレビューの下で、鍵への直接アクセスはなし

## ボットをインシデントにしない方法

ボットの恐怖譚話の多くは、同じ3つの過ちから生じます。出金権限付き API キー、二重支払いを引き起こすリトライループ、障害時の無音。私たちの標準プラクティス：

- ワークフローに必要な最小権限セットにスコープした取引所キー — 可能な限り取引専用または読み取り専用
- べき等な操作と明示的な状態管理により、再起動で送金が繰り返されない
- すべての外部 API 周辺にレート制限バジェットとサーキットブレーカー
- ボット自体のアラート — 無音で停止するボットは、ボットがないより悪い
- お客様のサーバーへのデプロイ。チームが読めるログとメトリクス

## 事例：ブロックチェーン検証 2FA

[adamant-2fa](https://github.com/Adamant-im/adamant-2fa) は、SMS の代わりに ADAMANT メッセンジャー経由でワンタイムコードを配信します。配信はオンチェーンで検証され、SIM スワップの対象になりません。メールや SMS が許容できない攻撃面となるプロダクト向けに、同様の検証フローを構築します。

## 事例：暗号化チャット内の取引所

[adamant-exchangebot](https://github.com/Adamant-im/adamant-exchangebot) は、見積もり、入金、ペイアウトを含む完全な取引所ワークフローを、エンドツーエンド暗号化チャット内で実行します。同じパターンは、ペイアウト、フォーセット、バウンティ支払い、自社スタック内の内部トークン配布に適応できます。

## 境界線

私たちはお客様に代わって戦略を実行せず、鍵を保持せず、トレード利益を約束しません。「お金を生み出すボットを作って」という依頼には、ノーと理由を説明し — チームのエンジニアリング時間を実際に節約する自動化を提案します。
