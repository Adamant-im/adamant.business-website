加密基础设施的失败方式与普通 Web 基础设施不同：节点停滞意味着漏收充值，错误升级可能让你脱离网络，而涉及资金时「从备份恢复」后果严重。我们为自有生态运行这类系统——基础设施合作通常按以下阶段推进。

## 阶段 0 — 评估

我们从阅读你现有环境开始：链、钱包、节点版本、托管方式、备份方案，以及你最担心的故障点。你会收到简短书面评估，列出具体风险与目标架构建议——即便合作在此止步也有参考价值。

## 阶段 1 — 部署

节点、浏览器、索引器、钱包后端与 API 部署在你的服务器或你控制的专用主机上。一切可复现：配置入库、文档化引导流程，没有只有某个承包商才懂的「雪花服务器」。

## 阶段 2 — 可观测性

在称某物「完成」之前，它必须能自我报告：区块高度滞后、对等节点数、磁盘余量、API 延迟、钱包余额阈值。告警进入你团队的渠道——Telegram、ADAMANT、邮件——并附带常见情况的运行手册。

## 阶段 3 — 运维

按计划应用链升级与硬分叉，修补依赖，审查容量。我们提供持续维护合同，或向你的团队干净移交——附文档与培训；自托管意味着你永远不会被锁定。

## 源自我们自用的组件

[ipfs-node](https://github.com/Adamant-im/ipfs-node) 是我们的分布式存储节点，用于不适合上链的文件。[currencyinfo](https://github.com/Adamant-im/currencyinfo) 是自托管的加密与法币汇率服务——生产系统默默依赖的那种枯燥而关键的基础组件。两者皆开源，且今日运行于我们自有基础设施。

## 我们坚持的标准

我们维护自己构建的系统。当你的节点、钱包与 API 需要在线运行数年——而不只是在路演中演示良好——这一点至关重要。
