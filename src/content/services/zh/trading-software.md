交易软件在最糟糕的时刻接受检验：交易所 API 降级、WebSocket 静默断开、重启中途出现部分成交。我们为这些时刻构建系统——自托管、可观测，从第一次提交起就在你团队掌控之下。

## 典型构建内容

- **交易所连接** — 针对你使用的 CEX 与 DEX 的 REST 与 WebSocket 连接器，含重连、时钟漂移处理及按场所的速率限制预算
- **订单引擎** — 可经受重启的订单生命周期跟踪：每笔订单与交易所对账，从不假设状态
- **风险护栏** — 硬性持仓上限、名义价值限制、紧急停止开关，以及介于策略代码与交易所之间的健全性检查
- **市场数据管道** — 跨场所归一化的成交、订单簿与 K 线，存储于分析师可查询的位置
- **运营视图** — 仪表盘与告警，让人工始终了解系统在做什么，并能一键停止

## 做市，具体而言

十余年运行自有做市软件的经验沉淀于 [adamant-tradebot](https://github.com/Adamant-im/adamant-tradebot)——专注做市与交易的软件，提供免费开源版，托管高级体验见 [marketmaking.app](https://marketmaking.app)。对需要在自有硬件上运行的团队——自定义交易对、场所与风险规则——我们基于同一引擎定制自托管部署。

## 资金与执行工具

并非一切都是策略。基金与产品公司找我们做的是不起眼的一层：跨场所再平衡、大额头寸 TWAP 式执行、批量付款，以及精确到聪的对账报告。这是自动化最快回本的地方。
